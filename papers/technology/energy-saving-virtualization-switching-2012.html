  • ការប្រើប្រាស់បច្ចេកទេស Dynamic Voltage and Frequency Scaling (DVFS) នៅក្នុង Network on Chip

  • ការប្រើប្រាស់ Virtualization អាចធ្វើឱ្យសមត្ថភាពបញ្ជូនទិន្នន័យថយចុះប្រហែល ៤០% បើធៀបនឹងការប្រើប្រាស់ Hardware ផ្ទាល់ ប៉ុន្តែការកែសម្រួលការកំណត់ (Tuning) អាចជួយកាត់បន្ថយផលប៉ះពាល់នេះបានយ៉ាងច្រើន។
  • ការអនុវត្តបច្ចេកទេស DVFS រួមជាមួយការថ្លឹងថ្លែងបន្ទុកចរាចរណ៍ (Load Balancing) នៅក្នុង Network on Chip (NoC) អាចសន្សំសំចៃថាមពលបានរហូតដល់ ៧៨% សម្រាប់គំរូចរាចរណ៍មួយចំនួន។
  • វិធីសាស្រ្តរចនាថ្មីសម្រាប់ Multistage Software Router (MSR) ដែលបានស្នើឡើងអាចកាត់បន្ថយការប្រើប្រាស់ថាមពលបានប្រហែល ១០% ទៅ ២០% បើធៀបនឹងវិធីសាស្រ្តដែលមានស្រាប់។

XY Routing without DVFS

Multiplane NoC with DVFS & Load Concentration (2P-4PHASE)
បណ្តាញលើបន្ទះឈីប (NoC) ពហុប្លង់ជាមួយ DVFS និងការប្រមូលផ្តុំបន្ទុក (Load Concentration)		 អនុញ្ញាតឱ្យប្លង់មួយដំណើរការនៅប្រេកង់ទាបបំផុតដើម្បីសន្សំសំចៃថាមពល ខណៈប្លង់មួយទៀតដោះស្រាយចរាចរណ៍ដែលមមាញឹក។ ត្រូវការធនធានផ្នែករឹងទ្វេដង (ពីរផ្លូវ/planes) និងមានភាពស្មុគស្មាញក្នុងការបែងចែកចរាចរណ៍។ សន្សំសំចៃថាមពលបានរហូតដល់ ៧៨% នៅក្នុងគំរូចរាចរណ៍ Transpose បើធៀបនឹងការមិនប្រើ DVFS។
Monolithic Virtual Software Router
រ៉ោតទ័រកម្មវិធីនិម្មិតតែមួយ (Monolithic VSR) ដំណើរការលើ KVM		 មានភាពងាយស្រួលក្នុងការគ្រប់គ្រង និងអាចផ្លាស់ទី (Migrate) បានយ៉ាងងាយស្រួល។ ជួបប្រទះបញ្ហាស្ទះ (Bottleneck) នៅពេលដំណើរការកញ្ចប់ទិន្នន័យតូចៗ ដោយសារការបែងចែកធនធាន CPU។ ល្បឿនបញ្ជូនទិន្នន័យមានកម្រិតត្រឹមប្រហែល ៩០០ kpps សម្រាប់កញ្ចប់ទិន្នន័យទំហំ ៦៤ បៃ។
Goal Programming MSR Design

  2. ដំឡើងបរិស្ថានពិសោធន៍: ដំឡើងប្រព័ន្ធប្រតិបត្តិការ Linux (Ubuntu) និងដំឡើង (KVM) ព្រមទាំង (Click Modular Router) ដើម្បីបង្កើតជា Software Router លើកុំព្យូទ័រផ្ទាល់ខ្លួន។
  3. អនុវត្តការវាស់វែងសមត្ថភាព: ប្រើប្រាស់ឧបករណ៍ដូចជា (iperf) ឬ (pktgen) ដើម្បីវាស់ល្បឿនបញ្ជូនទិន្នន័យ (Throughput) និងសិក្សាពីផលប៉ះពាល់នៃការកំណត់ CPU Affinity ដូចបានរៀបរាប់ក្នុងជំពូកទី ៤។
  4. ពិសោធន៍ជាមួយការសន្សំសំចៃថាមពល: សាកល្បងសរសេរ Script សាមញ្ញដើម្បីបិទ/បើក Virtual Machines តាមតម្រូវការចរាចរណ៍ជាក់ស្តែង ឬប្រើប្រាស់កម្មវិធីត្រាប់តាមដើម្បីសិក្សាពីប្រសិទ្ធភាពនៃ DVFS។
