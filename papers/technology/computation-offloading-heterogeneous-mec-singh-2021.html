១. សេចក្តីសង្ខេបប្រតិបត្តិ (Executive Summary)

បញ្ហា (The Problem)៖ ការស្រាវជ្រាវនេះដោះស្រាយបញ្ហាដែនកំណត់នៃថាមពលថ្ម និងល្បឿនគណនារបស់ឧបករណ៍ចល័ត ព្រមទាំងដោះស្រាយបញ្ហាភាពយឺតយ៉ាវ (High Latency) ដែលកើតមាននៅក្នុងការគណនាលើ Cloud បែបបុរាណ ដោយងាកមកប្រើប្រាស់បច្ចេកវិទ្យាគណនានៅគែមបណ្ដាញ (MEC) វិញ។

វិធីសាស្ត្រ (The Methodology)៖ ការសិក្សានេះប្រើប្រាស់ការបង្កើតគំរូគណិតវិទ្យា និងការពិសោធន៍តាមរយៈកុំព្យូទ័រ ដើម្បីវិភាគកត្តាដែលជះឥទ្ធិពលដល់ការបន្ធូរបន្ថយបន្ទុកកុំព្យូទ័រ (Computation Offloading) និងបង្កើតក្បួនដោះស្រាយដើម្បីបង្កើនប្រសិទ្ធភាព។

  • Mathematical Modeling (ការបង្កើតគំរូគណិតវិទ្យា) ដើម្បីវិភាគពេលវេលា និងការប្រើប្រាស់ថាមពល
  • Linear Programming with CPLEX (ការសរសេរកម្មវិធីលីនេអ៊ែរជាមួយ CPLEX) ដើម្បីស្វែងរកដំណោះស្រាយប្រសើរបំផុតសម្រាប់ធ្វើការប្រៀបធៀប
  • Heuristic Algorithms (ក្បួនដោះស្រាយ Heuristic) សម្រាប់ការរៀបចំកាលវិភាគនៃការបញ្ជូនទិន្នន័យពីឧបករណ៍មួយឬច្រើន

លទ្ធផលសំខាន់ៗ (The Verdict)៖

  • ការបន្ធូរបន្ថយបន្ទុកកុំព្យូទ័រ (Computation Offloading) ផ្តល់ផលចំណេញលើពេលវេលា និងថាមពលបាន លុះត្រាតែមានល្បឿនបណ្ដាញទំនាក់ទំនងខ្ពស់ និងល្បឿនកែច្នៃទិន្នន័យរបស់ Server លឿនជាងឧបករណ៍ចល័តខ្លាំង។
  • ក្បួនដោះស្រាយ Heuristic ដែលបានបង្កើតឡើងអាចផ្តល់លទ្ធផលល្អប្រសើរដោយមានគម្លាតត្រឹមតែ ១% ទៅ ២០% ពីដំណោះស្រាយល្អបំផុត (Optimal solution) ដែលរកឃើញដោយ Linear Programming ប៉ុន្តែចំណាយពេលគណនាតិចជាង។
  • ការវិភាគពហុគោលបំណងបង្ហាញថា មិនមានដំណោះស្រាយសកលដែលល្អបំផុតទាំងពេលវេលា និងថាមពលព្រមគ្នានោះទេ គឺតម្រូវឱ្យមានការដោះដូរ (Trade-off) រវាងការកាត់បន្ថយពេលវេលាបញ្ចប់ការងារ និងការសន្សំសំចៃថាមពលថ្ម។
