Long Short-Term Memory (LSTM)
បណ្តាញសតិរយៈពេលវែង-ខ្លី (LSTM)		 មានប្រសិទ្ធភាពខ្ពស់ក្នុងការវិភាគទិន្នន័យតាមពេលវេលា (Time-series) និងចាប់យកភាពអាស្រ័យនៃលំនាំទិន្នន័យដើម្បីទស្សន៍ទាយការខូចខាត។ ទាមទារថាមពលកុំព្យូទ័រខ្លាំងសម្រាប់ការបណ្តុះបណ្តាល និងងាយប្រឈមនឹងបញ្ហា Overfitting ប្រសិនបើទិន្នន័យមានកម្រិត។ ពិន្ទុ F1 ០.៩២ និងកាត់បន្ថយពេលវេលាគាំងម៉ាស៊ីន (Downtime) បាន ២០%។
Convolutional Neural Networks (CNNs)
បណ្តាញសរសៃប្រសាទ Convolutional (CNNs)		 ពូកែក្នុងការរកមើលភាពមិនប្រក្រតីពីទិន្នន័យដែលមានវិមាត្រខ្ពស់ ដូចជារូបភាពកម្ដៅ ឬ Spectrograms នៃរំញ័រម៉ាស៊ីន។ ទាមទារការរៀបចំទិន្នន័យ (Preprocessing) ច្រើន និងមិនសូវមានប្រសិទ្ធភាពចំពោះការវិភាគដែលត្រូវការលំដាប់ពេលវេលាវែង។ ពិន្ទុ F1 ០.៩៣ និងភាពជាក់លាក់ (Precision) ០.៩៥ ក្នុងការរកឃើញភាពមិនប្រក្រតី។
Reinforcement Learning (RL)
ការរៀនពង្រឹង (Reinforcement Learning)		 មានសមត្ថភាពធ្វើការសម្រេចចិត្តបែបឌីណាមិកក្នុងពេលជាក់ស្តែង (Real-time) ដើម្បីរក្សាលំនឹងបណ្តាញអគ្គិសនី។ ត្រូវការការពិសោធន៍រាប់ពាន់ដងក្នុងបរិយាកាសនិម្មិតដើម្បីរៀន និងប្រឈមនឹងការលំបាកក្នុងការកំណត់ Reward function ឱ្យត្រឹមត្រូវ។ កាត់បន្ថយគម្លាតប្រេកង់បណ្តាញបាន ៣០% និងបង្កើនប្រសិទ្ធភាពផ្ទុកថាមពល ២៥%។

៦. ផែនការសកម្មភាពសម្រាប់និស្សិត (Actionable Roadmap)

ដើម្បីអនុវត្តតាមការសិក្សានេះ និស្សិតគួរអនុវត្តតាមជំហានខាងក្រោម៖

  1. ការសិក្សាមូលដ្ឋានគ្រឹះ និងឧបករណ៍: និស្សិតគួរចាប់ផ្តើមរៀនភាសា Python និងបណ្ណាល័យសំខាន់ៗដូចជា TensorFlow ឬ PyTorch។ ផ្តោតលើការយល់ដឹងពីរបៀបបង្កើតម៉ូដែល LSTM និង CNN សម្រាប់ការវិភាគ Time-series។
  2. ការប្រមូល និងសម្អាតទិន្នន័យ: រៀនប្រើប្រាស់ Scikit-learn ដើម្បីធ្វើ Data Preprocessing ដូចជាការបំពេញទិន្នន័យដែលបាត់ (Imputation) និងការធ្វើ Normalization។ អាចសាកល្បងរក Dataset ពី Kaggle ដែលទាក់ទងនឹង 'Wind turbine data' ឬ 'Solar power generation'។
  3. ការអភិវឌ្ឍ និងបណ្តុះបណ្តាលម៉ូដែល: សាកល្បងបង្កើតគម្រោងតូចមួយដើម្បីទស្សន៍ទាយការខូចខាតឧបករណ៍ (Predictive Maintenance) ដោយប្រើទិន្នន័យប្រវត្តិ។ ប្រើប្រាស់ Google Colab ប្រសិនបើគ្មានកុំព្យូទ័រដែលមាន GPU ខ្លាំង។
  4. ការពិសោធន៍ជាមួយ Reinforcement Learning: សិក្សាពីមូលដ្ឋាននៃ RL ដោយប្រើប្រាស់ OpenAI Gym ដើម្បីបង្កើតបរិយាកាសពិសោធន៍ (Simulation) សម្រាប់ការគ្រប់គ្រងបណ្តាញថាមពលខ្នាតតូច។

៧. វាក្យសព្ទបច្ចេកទេស (Technical Glossary)

ពាក្យបច្ចេកទេស (English) ការពន្យល់ជាខេមរភាសា (Khmer Explanation) និយមន័យសាមញ្ញ (Simple Definition)
Predictive Maintenance ជាយុទ្ធសាស្ត្រថែទាំដែលប្រើប្រាស់ការវិភាគទិន្នន័យដើម្បីត្រួតពិនិត្យស្ថានភាពជាក់ស្តែងនៃឧបករណ៍ និងព្យាករណ៍ថាពេលណាឧបករណ៍នោះនឹងខូច ដើម្បីធ្វើការជួសជុលមុនពេលមានបញ្ហាកើតឡើង។ ដូចជាការពិនិត្យសុខភាពរាងកាយដើម្បីការពារជំងឺធ្ងន់ធ្ងរ ជាជាងរង់ចាំទាល់តែឈឺធ្ងន់ទើបទៅជួបពេទ្យ។
Long Short-Term Memory (LSTM) ជាប្រភេទនៃបណ្តាញសរសៃប្រសាទសិប្បនិម្មិត (AI) ដែលមានសមត្ថភាពចងចាំព័ត៌មានពីអតីតកាលក្នុងរយៈពេលវែង។ វាមានប្រសិទ្ធភាពខ្ពស់ក្នុងការវិភាគទិន្នន័យដែលមានលំដាប់លំដោយ ដូចជាទិន្នន័យប្រើប្រាស់អគ្គិសនីប្រចាំថ្ងៃ។ ប្រៀបដូចជាការអានសៀវភៅ ដែលអ្នកត្រូវចងចាំសាច់រឿងពីទំព័រមុនៗ ដើម្បីយល់ពីព្រឹត្តិការណ៍បច្ចុប្បន្ន។
Convolutional Neural Networks (CNNs) ជាក្បួនដោះស្រាយ AI ដែលត្រូវបានរចនាឡើងយ៉ាងពិសេសដើម្បីវិភាគទិន្នន័យជារូបភាព ឬក្រឡា (Grid)។ ក្នុងឯកសារនេះ វាត្រូវបានប្រើដើម្បីវិភាគរូបភាពកម្ដៅ ឬក្រាហ្វិកនៃរំញ័រម៉ាស៊ីន ដើម្បីរកមើលភាពមិនប្រក្រតី។ ដូចជាការពាក់វ៉ែនតាពិសេសដែលអាចស្កេនរកមើលស្នាមប្រេះ ឬចំណុចក្ដៅខុសធម្មតានៅលើម៉ាស៊ីនបានភ្លាមៗ។
Reinforcement Learning (RL) ជាវិធីសាស្ត្របង្រៀនម៉ាស៊ីន (Machine Learning) ដែលឱ្យ AI រៀនធ្វើការសម្រេចចិត្តដោយស្វ័យប្រវត្តិ តាមរយៈការសាកល្បងខុសត្រូវ (Trial and Error) នៅក្នុងបរិយាកាសនិម្មិត ដើម្បីទទួលបានលទ្ធផលល្អបំផុត។ ដូចការបង្រៀនសត្វចិញ្ចឹម ដោយផ្តល់រង្វាន់នៅពេលវាធ្វើត្រូវ និងមិនផ្តល់រង្វាន់នៅពេលវាធ្វើខុស រហូតទាល់តែវាចេះធ្វើដោយខ្លួនឯង។
Spectrograms ជាការបង្ហាញរូបភាពនៃរលកសញ្ញា (ដូចជាសំឡេង ឬរំញ័រ) ដែលប្រែប្រួលតាមពេលវេលា។ ឯកសារនេះប្រើវាដើម្បីបំប្លែងទិន្នន័យរំញ័ររបស់ទួរប៊ីនខ្យល់ ទៅជារូបភាព ដើម្បីឱ្យម៉ូដែល CNN អាចវិភាគបាន។ ដូចជាការមើលឃើញរូបរាងនៃសំឡេងនៅលើអេក្រង់ម៉ាស៊ីនចាក់តន្ត្រី ជាជាងគ្រាន់តែស្តាប់ដោយត្រចៀក។
Grid Stability សមត្ថភាពនៃប្រព័ន្ធអគ្គិសនីក្នុងការរក្សាតុល្យភាពរវាងការផលិត និងការប្រើប្រាស់ថាមពល ព្រមទាំងរក្សាប្រេកង់ (Frequency) ឱ្យនៅថេរ ទោះបីជាមានការប្រែប្រួលនៃពន្លឺព្រះអាទិត្យ ឬកម្លាំងខ្យល់ក៏ដោយ។ ប្រៀបដូចជាការដើរទូលទឹកមួយថាស អ្នកត្រូវរក្សាលំនឹងខ្លួនជានិច្ចដើម្បីកុំឱ្យទឹកកំពប់ ទោះបីជាផ្លូវរលាក់ក៏ដោយ។
