អ្នកនិពន្ធ៖ Sreelakshmi Koratagere (RV College of Engineering), Ravi Kumar Chandrashekarappa Koppal (RV College of Engineering), Iyyanahalli Math Umesh (RV College of Engineering)

១. សេចក្តីសង្ខេបប្រតិបត្តិ (Executive Summary)

បញ្ហា (The Problem)៖ ការសិក្សានេះដោះស្រាយបញ្ហាប្រសិទ្ធភាពនៃការប្រើប្រាស់ហេដ្ឋារចនាសម្ព័ន្ធបច្ចេកវិទ្យាក្នុងសាកលវិទ្យាល័យ ជាពិសេសការចំណាយលើថាមពលអគ្គិសនីខ្ពស់ ការប្រើប្រាស់ផ្នែករឹង (Hardware) មិនអស់លទ្ធភាព និងការចំណាយលើការថែទាំខ្ពស់ដែលបណ្តាលមកពីការប្រើប្រាស់ម៉ាស៊ីនមេ (Servers) ដាច់ដោយឡែកពីគ្នាជាច្រើន។

វិធីសាស្ត្រ (The Methodology)៖ ការសិក្សានេះបង្ហាញពីដំណើរការផ្លាស់ប្តូរពីការប្រើប្រាស់ម៉ាស៊ីនមេដាច់ដោយឡែក ទៅកាន់បរិស្ថាននិម្មិត (Virtual environment) ដោយអនុវត្តតាមជំហានជាប្រព័ន្ធ។

  • ការវាយតម្លៃ និងការជ្រើសរើសផលិតផលនិម្មិតកម្ម (Virtualization Product Evaluation) ដូចជា VMware, Citrix និង Microsoft ដើម្បីធ្វើតេស្តសាកល្បង (Proof of Concept)។
  • ការផ្លាស់ប្តូរពីម៉ាស៊ីនរូបវន្តទៅជាម៉ាស៊ីននិម្មិត (Physical-to-Virtual Migration - P2V) និងការដាក់ឱ្យប្រើប្រាស់ហេដ្ឋារចនាសម្ព័ន្ធ Hyper-Converged Infrastructure (HCI)។

លទ្ធផលសំខាន់ៗ (The Verdict)៖

  • ការកាត់បន្ថយចំនួនម៉ាស៊ីនមេរូបវន្ត (Physical Servers) យ៉ាងច្រើនពី ៣៦ គ្រឿង មកនៅត្រឹម ៣ គ្រឿងប៉ុណ្ណោះនៅក្នុងមជ្ឈមណ្ឌលទិន្នន័យ។
  • ការសន្សំសំចៃថាមពលអគ្គិសនីបានយ៉ាងល្អប្រសើរ ដោយការប្រើប្រាស់ថាមពលសរុបបានធ្លាក់ចុះពី ៥៧៦០ វ៉ាត់ មកនៅត្រឹម ១២៧៥ វ៉ាត់។
  • ការបង្កើនប្រសិទ្ធភាពនៃការប្រើប្រាស់ធនធាន CPU និង Memory ព្រមទាំងផ្តល់លទ្ធភាពងាយស្រួលក្នុងការគ្រប់គ្រង និងពង្រីកប្រព័ន្ធសម្រាប់ការស្រាវជ្រាវ។

២. ការវិភាគលើប្រសិទ្ធភាព និងការចំណាយ (Performance & Constraints)

វិធីសាស្ត្រ (Method) គុណសម្បត្តិ (Pros) គុណវិបត្តិ (Cons) លទ្ធផលគន្លឹះ (Key Result)
Traditional Physical Infrastructure
ហេដ្ឋារចនាសម្ព័ន្ធរូបវន្តបែបប្រពៃណី (ម៉ាស៊ីនមេដាច់ដោយឡែក)		 មានភាពងាយស្រួលក្នុងការដាក់ឱ្យដំណើរការដំបូងសម្រាប់កម្មវិធីតែមួយមុខ និងមិនត្រូវការចំណេះដឹងផ្នែកនិម្មិតកម្ម (Virtualization) ខ្ពស់។ ការប្រើប្រាស់ធនធានមិនអស់លទ្ធភាព (ប្រហែល ២០% នៃសមត្ថភាពម៉ាស៊ីន) និងចំណាយលើថាមពលអគ្គិសនីខ្ពស់ខ្លាំង។ ប្រើប្រាស់ម៉ាស៊ីនមេចំនួន ៣៦ គ្រឿង និងប្រើប្រាស់ថាមពលសរុប ៥៧៦០ វ៉ាត់។
Server Virtualization (HCI)
និម្មិតកម្មម៉ាស៊ីនមេ (ប្រើប្រាស់បច្ចេកវិទ្យា Hyper-Converged)		 ផ្តល់នូវលទ្ធភាពធ្វើមាត្រដ្ឋាន (Scalability) ខ្ពស់ ការគ្រប់គ្រងមជ្ឈការ និងសមត្ថភាពបម្រុងទុកទិន្នន័យ (Backup) បានល្អប្រសើរ។ ទាមទារឱ្យមានការវិនិយោគលើផ្នែករឹង (Hardware) ដែលមានសមត្ថភាពខ្ពស់នៅដំណាក់កាលដំបូង និងជំនាញបច្ចេកទេសច្បាស់លាស់។ កាត់បន្ថយមកត្រឹម ៣ គ្រឿង (Nodes) និងប្រើប្រាស់ថាមពលត្រឹម ១២៧៥ វ៉ាត់ (សន្សំសំចៃបានប្រហែល ៧៨%)។

ការចំណាយលើធនធាន (Resource Cost)៖ ការអនុវត្តគម្រោងនេះទាមទារការវិនិយោគលើឧបករណ៍ផ្នែករឹងដែលមានសមត្ថភាពខ្ពស់ដើម្បីជំនួសម៉ាស៊ីនចាស់ៗ ប៉ុន្តែផ្តល់ផលចំណេញរយៈពេលវែងលើថ្លៃអគ្គិសនី។

  • Hardware: ម៉ាស៊ីនមេដែលមានសមត្ថភាពខ្ពស់ (High-performance rack servers) និងឧបករណ៍ផ្ទុកទិន្នន័យ (NAS) សម្រាប់ដំណាក់កាលដំបូង។
  • Software: កម្មវិធី Hypervisor ដូចជា (VMware vSphere), (Microsoft Hyper-V), ឬ (Citrix XenServer)។ ក្នុងករណីនេះប្រើប្រាស់ VMware។
  • Expertise: អ្នកបច្ចេកទេស IT ដែលមានជំនាញក្នុងការធ្វើ (P2V Migration) និងការគ្រប់គ្រងប្រព័ន្ធ Cluster។

៣. ការពិនិត្យសម្រាប់បរិបទកម្ពុជា/អាស៊ីអាគ្នេយ៍

ភាពលំអៀងនៃទិន្នន័យ (Data Bias)៖

ការសិក្សានេះធ្វើឡើងនៅមហាវិទ្យាល័យវិស្វកម្ម RV ក្នុងប្រទេសឥណ្ឌា ដែលមានបរិបទប្រហាក់ប្រហែលនឹងគ្រឹះស្ថានឧត្តមសិក្សានៅកម្ពុជា ទាក់ទងនឹងតម្រូវការកាត់បន្ថយចំណាយប្រតិបត្តិការ និងការប្រើប្រាស់ធនធានដែលមានកម្រិត។ ទោះជាយ៉ាងណាក៏ដោយ ការសិក្សានេះប្រើប្រាស់ផលិតផលដែលមានអាជ្ញាប័ណ្ណ (VMware) ដែលអាចមានតម្លៃខ្ពស់សម្រាប់សាកលវិទ្យាល័យមួយចំនួន។

លទ្ធភាពនៃការអនុវត្ត (Applicability)៖

បច្ចេកវិទ្យានេះមានសារៈសំខាន់ និងអាចអនុវត្តបានយ៉ាងពេញលេញសម្រាប់កម្ពុជា ជាពិសេសក្នុងបរិបទដែលតម្លៃអគ្គិសនីមានកម្រិតខ្ពស់។

  • គ្រឹះស្ថានឧត្តមសិក្សារដ្ឋ និងឯកជន: សាកលវិទ្យាល័យដូចជា RUPP ឬ ITC អាចកាត់បន្ថយបន្ទប់ Server នៅតាមដេប៉ាតឺម៉ង់នីមួយៗមកជាមជ្ឈមណ្ឌលទិន្នន័យរួមមួយ (Centralized Data Center) ដើម្បីងាយស្រួលគ្រប់គ្រង និងសុវត្ថិភាព។
  • វិស័យធនាគារ និងមីក្រូហិរញ្ញវត្ថុ: ការប្រើប្រាស់ (Virtualization) ជួយសម្រួលដល់ការធ្វើ (Disaster Recovery) ដែលជាតម្រូវការចាំបាច់សម្រាប់សុវត្ថិភាពទិន្នន័យហិរញ្ញវត្ថុ។
  • ក្រសួង និងស្ថាប័នរដ្ឋ: គាំទ្រដល់គោលនយោបាយរដ្ឋាភិបាលឌីជីថល ដោយបង្កើត Private Cloud សម្រាប់ផ្ទុកទិន្នន័យប្រកបដោយប្រសិទ្ធភាព និងសន្សំសំចៃ។

ការធ្វើនិម្មិតកម្មម៉ាស៊ីនមេ មិនត្រឹមតែជួយសន្សំសំចៃថវិកាលើថ្លៃអគ្គិសនីប៉ុណ្ណោះទេ ប៉ុន្តែថែមទាំងបង្កើនសមត្ថភាពនៃការស្រាវជ្រាវ និងការបង្រៀនតាមរយៈការផ្តល់ធនធាន IT បានរហ័ស។

៤. ផែនការសកម្មភាពសម្រាប់និស្សិត (Actionable Roadmap)

ដើម្បីអនុវត្តតាមការសិក្សានេះ និស្សិតគួរអនុវត្តតាមជំហានខាងក្រោម៖

  1. ជំហានទី ១៖ ការវាយតម្លៃ និងសវនកម្ម (Assessment & Audit): ធ្វើបញ្ជីសារពើភណ្ឌនៃម៉ាស៊ីនមេដែលមានស្រាប់ទាំងអស់ ដោយវាស់វែងកម្រិតនៃការប្រើប្រាស់ CPU និង RAM ជាក់ស្តែង ដើម្បីកំណត់ថាមានម៉ាស៊ីនប៉ុន្មានដែលដំណើរការក្រោមសមត្ថភាព (Underutilized)។
  2. ជំហានទី ២៖ ការជ្រើសរើសបច្ចេកវិទ្យា (Platform Selection): វាយតម្លៃលើកម្មវិធី Hypervisor ដូចជា (VMware), (Microsoft Hyper-V) ឬជម្រើស Open Source ដូចជា (Proxmox VE) ដើម្បីសន្សំថ្លៃអាជ្ញាប័ណ្ណ។
  3. ជំហានទី ៣៖ ការធ្វើតេស្តសាកល្បង (Proof of Concept - PoC): ជ្រើសរើសសេវាកម្មដែលមិនសូវសំខាន់ (Non-critical services) ដើម្បីធ្វើការសាកល្បងផ្លាស់ប្តូរទៅកាន់ប្រព័ន្ធនិម្មិត និងតាមដានស្ថេរភាពនៃប្រព័ន្ធ។
  4. ជំហានទី ៤៖ ការផ្លាស់ប្តូរ និងការបង្រួម (Migration & Consolidation): អនុវត្តការផ្លាស់ប្តូរពីម៉ាស៊ីនរូបវន្តទៅម៉ាស៊ីននិម្មិត (P2V Migration) និងបិទម៉ាស៊ីនមេដែលលែងប្រើប្រាស់ ដើម្បីកាត់បន្ថយការចំណាយលើថាមពលអគ្គិសនី។

៥. វាក្យសព្ទបច្ចេកទេស (Technical Glossary)

ពាក្យបច្ចេកទេស (English) ការពន្យល់ជាខេមរភាសា (Khmer Explanation) និយមន័យសាមញ្ញ (Simple Definition)
Server Virtualization បច្ចេកវិទ្យាដែលអនុញ្ញាតឱ្យម៉ាស៊ីនមេ (Server) មួយគ្រឿងអាចដំណើរការប្រព័ន្ធប្រតិបត្តិការ (OS) ច្រើនក្នុងពេលតែមួយ ដោយបែងចែកធនធានផ្នែករឹងឱ្យទៅម៉ាស៊ីននិម្មិត (Virtual Machines) នីមួយៗដាច់ដោយឡែកពីគ្នា។ វាជួយឱ្យប្រើប្រាស់ធនធានម៉ាស៊ីនបានអស់លទ្ធភាព។ ដូចជាការបែងចែកផ្ទះធំមួយឱ្យទៅជាបន្ទប់ជួលជាច្រើន ដែលបន្ទប់នីមួយៗមានអ្នករស់នៅរៀងៗខ្លួន ប៉ុន្តែប្រើប្រាស់ទឹកភ្លើងនិងដំបូលរួមគ្នា។
Hypervisor ជាកម្មវិធីកុំព្យូទ័រពិសេស (Software Layer) ដែលដើរតួជាអ្នកគ្រប់គ្រងនៅចន្លោះផ្នែករឹង (Hardware) និងម៉ាស៊ីននិម្មិត (Virtual Machines)។ វាមានតួនាទីបែងចែក CPU, RAM និង Storage ទៅឱ្យម៉ាស៊ីននិម្មិតនីមួយៗ និងធានាថាពួកវាមិនរំខានគ្នា។ ដូចជាប៉ូលិសចរាចរណ៍នៅផ្លូវបំបែក ដែលចាត់ចែងថាឡានណាត្រូវទៅមុន ឡានណាត្រូវឈប់ ដើម្បីកុំឱ្យមានការកកស្ទះ។
Hyper Converged Infrastructure (HCI) ជាប្រព័ន្ធហេដ្ឋារចនាសម្ព័ន្ធ IT ទំនើបដែលរួមបញ្ចូលគ្នានូវផ្នែកគណនា (Compute) ការផ្ទុកទិន្នន័យ (Storage) និងបណ្តាញ (Networking) ចូលទៅក្នុងឧបករណ៍តែមួយ ដោយប្រើប្រាស់កម្មវិធី (Software) ដើម្បីគ្រប់គ្រង ជំនួសឱ្យការទិញឧបករណ៍ដាច់ដោយឡែកពីគ្នា។ ដូចជាទូរស័ព្ទស្មាតហ្វូនដែលបញ្ចូលគ្នានូវកាមេរ៉ា ម៉ាស៊ីនគិតលេខ និងទូរស័ព្ទទៅក្នុងឧបករណ៍តែមួយ ជំនួសឱ្យការកាន់ឧបករណ៍បីផ្សេងគ្នា។
P2V (Physical to Virtual) ដំណើរការបច្ចេកទេសនៃការចម្លងប្រព័ន្ធប្រតិបត្តិការ កម្មវិធី និងទិន្នន័យទាំងអស់ពីកុំព្យូទ័ររូបវន្ត (Physical Machine) ទៅជាទម្រង់ឌីជីថល (Virtual Machine) ដើម្បីឱ្យវាអាចដំណើរការលើបរិស្ថាននិម្មិត (Cloud/Virtual Environment) បាន។ ដូចជាការយកសៀវភៅក្រដាសទៅស្កេន (Scan) ដើម្បីប្រែក្លាយវាទៅជាឯកសារ PDF ក្នុងកុំព្យូទ័រ។
Thin Provisioning បច្ចេកទេសគ្រប់គ្រងការផ្ទុកទិន្នន័យដែលអនុញ្ញាតឱ្យយើងបង្កើតទំហំផ្ទុកនិម្មិតធំជាងទំហំរូបវន្តជាក់ស្តែង។ ទំហំផ្ទុកជាក់ស្តែងនឹងត្រូវកាត់យកមកប្រើប្រាស់តែនៅពេលដែលមានការសរសេរទិន្នន័យចូលប៉ុណ្ណោះ ដែលជួយសន្សំសំចៃទំហំ Hard Disk។ ដូចជាធនាគារផ្តល់កាតឥណទានដែលមានទំហំទឹកប្រាក់ ១ ម៉ឺនដុល្លារ ប៉ុន្តែធនាគារមិនចាំបាច់ដកលុយសុទ្ធ ១ ម៉ឺនដុល្លារមកទុកនោះទេ គឺដកតែចំនួនដែលអតិថិជនចាយជាក់ស្តែងប៉ុណ្ណោះ។
Server Consolidation យុទ្ធសាស្ត្រនៃការកាត់បន្ថយចំនួនម៉ាស៊ីនមេរូបវន្ត (Physical Servers) ដោយប្រមូលផ្តុំការងារពីម៉ាស៊ីនច្រើនឱ្យមកដំណើរការនៅលើម៉ាស៊ីនមេដែលមានសមត្ថភាពខ្ពស់មួយចំនួនតូចវិញ ដើម្បីកាត់បន្ថយការចំណាយលើការថែទាំនិងអគ្គិសនី។ ដូចជាការជិះឡានក្រុងរួមគ្នាជំនួសឱ្យមនុស្សម្នាក់ៗជិះឡានផ្ទាល់ខ្លួនម្នាក់មួយគ្រឿង ដើម្បីសន្សំសាំង និងកាត់បន្ថយការស្ទះផ្លូវ។
