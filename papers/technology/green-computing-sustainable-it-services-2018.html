ចំណងជើងដើម៖ Assessing the Influence of Green Computing Practices on Sustainable IT Services

អ្នកនិពន្ធ៖ Shedrack Mmeah (Ken Saro Wiwa Polytechnic), Barida Baah (Ebonyi State University), Abasiama G. Akpan (Ebonyi State University)

វិស័យសិក្សា៖ Computer Science / IT Management

១. សេចក្តីសង្ខេបប្រតិបត្តិ (Executive Summary)

បញ្ហា (The Problem)៖ ការសិក្សានេះដោះស្រាយបញ្ហានៃការកើនឡើងនូវការប្រើប្រាស់ថាមពល និងផលប៉ះពាល់បរិស្ថានដែលបណ្តាលមកពីហេដ្ឋារចនាសម្ព័ន្ធព័ត៌មានវិទ្យា ជាពិសេសមជ្ឈមណ្ឌលទិន្នន័យ (Data Centers) ដែលទាមទារឱ្យមានការផ្លាស់ប្តូរពីការកាត់បន្ថយការចំណាយសាមញ្ញ ទៅរកសេវាកម្ម IT ប្រកបដោយនិរន្តរភាព។

វិធីសាស្ត្រ (The Methodology)៖ អ្នកនិពន្ធបានធ្វើការសិក្សាស្រាវជ្រាវដោយផ្អែកលើការពិនិត្យឯកសារ (Literature Review) ដើម្បីកំណត់និន្នាការបច្ចុប្បន្ន យុទ្ធសាស្ត្រសំខាន់ៗ និងគោលការណ៍សម្រាប់ការរចនាសេវាកម្ម IT។

  • Literature Review (ការពិនិត្យឯកសារស្រាវជ្រាវស្តីពីកុំព្យូទ័របៃតង)
  • Analysis of Metrics (ការវិភាគលើរង្វាស់ប្រសិទ្ធភាពដូចជា PUE និង DCiE)
  • Strategic Framework Assessment (ការវាយតម្លៃក្របខ័ណ្ឌយុទ្ធសាស្ត្រសម្រាប់ការផ្លាស់ប្តូរទៅកាន់និរន្តរភាព)

លទ្ធផលសំខាន់ៗ (The Verdict)៖

  • ការចំណាយលើថាមពលសម្រាប់ផ្នែក IT អាចកើនឡើងដល់ ៥០% នៃការចំណាយថាមពលសរុបរបស់ស្ថាប័ន ដែលធ្វើឱ្យប្រសិទ្ធភាពនៃមជ្ឈមណ្ឌលទិន្នន័យក្លាយជាបញ្ហាយុទ្ធសាស្ត្រដ៏សំខាន់។
  • ការអនុវត្តយុទ្ធសាស្ត្រដូចជា និម្មិតកម្ម (Virtualization) ការគណនាលើពពក (Cloud Computing) និងការគ្រប់គ្រងថាមពល ជួយបង្កើនប្រសិទ្ធភាពធនធាន និងកាត់បន្ថយការបំភាយកាបូន។
  • មានការផ្លាស់ប្តូរពីការផ្តោតលើការកាត់បន្ថយការចំណាយតែមួយមុខ (Green IT) ទៅជាការបង្កើតតម្លៃសម្រាប់សង្គម និងការទទួលខុសត្រូវសង្គមរបស់សាជីវកម្ម (Sustainable IT Services) ដែលទាមទារឱ្យមានការរៀបចំរចនាសម្ព័ន្ធស្ថាប័នឡើងវិញ។

២. ការវិភាគលើប្រសិទ្ធភាព និងការចំណាយ (Performance & Constraints)

វិធីសាស្ត្រ (Method) គុណសម្បត្តិ (Pros) គុណវិបត្តិ (Cons) លទ្ធផលគន្លឹះ (Key Result)
Traditional Data Center Hosting
ការផ្ទុកទិន្នន័យក្នុងមជ្ឈមណ្ឌលទិន្នន័យតាមបែបប្រពៃណី (មិនមាន Virtualization)		 មានការគ្រប់គ្រងពេញលេញលើផ្នែករឹង (Hardware) និងសុវត្ថិភាពផ្ទាល់ខ្លួន។ ការប្រើប្រាស់ធនធានមិនអស់លទ្ធភាព (Underutilization) និងការចំណាយលើថាមពលខ្ពស់។ អត្រាប្រើប្រាស់ Server មានត្រឹមតែ ៥-១០% ប៉ុណ្ណោះ ដែលធ្វើឱ្យខាតបង់ថាមពលច្រើន។
Virtualization
បច្ចេកវិទ្យានិម្មិតកម្ម (Virtualization)		 អនុញ្ញាតឱ្យដំណើរការប្រព័ន្ធប្រតិបត្តិការច្រើននៅលើម៉ាស៊ីនតែមួយ កាត់បន្ថយចំនួន Server និងការប្រើប្រាស់ថាមពល។ ទាមទារឱ្យមានជំនាញបច្ចេកទេសខ្ពស់ក្នុងការគ្រប់គ្រង Hypervisor និងការរៀបចំប្រព័ន្ធ។ អាចបង្កើនអត្រាប្រើប្រាស់ Server ដល់ ៥០-៨៥% និងកាត់បន្ថយតម្រូវការផ្នែករឹង។
Cloud Computing (SaaS, PaaS, IaaS)
ការគណនាលើពពក (Cloud Computing)		 មិនត្រូវការការវិនិយោគធំដំបូងលើហេដ្ឋារចនាសម្ព័ន្ធ (Upfront investment) និងអាចពង្រីកទំហំប្រើប្រាស់តាមតម្រូវការ។ ការពឹងផ្អែកលើការតភ្ជាប់អ៊ីនធឺណិត និងបញ្ហាភាពឯកជននៃទិន្នន័យប្រសិនបើមិនបានគ្រប់គ្រងត្រឹមត្រូវ។ កាត់បន្ថយការចំណាយប្រតិបត្តិការ និងការប្រើប្រាស់ថាមពល ដោយបង់ប្រាក់តាមការប្រើប្រាស់ជាក់ស្តែង។

ការចំណាយលើធនធាន (Resource Cost)៖ ការអនុវត្ត Green Computing ទាមទារការវិនិយោគលើបច្ចេកវិទ្យាគ្រប់គ្រងថាមពល និងការផ្លាស់ប្តូរហេដ្ឋារចនាសម្ព័ន្ធ ប៉ុន្តែផ្តល់ផលចំណេញរយៈពេលវែងតាមរយៈការកាត់បន្ថយថ្លៃអគ្គិសនី។

  • Hardware: ត្រូវការឧបករណ៍ដែលមានប្រសិទ្ធភាពថាមពលខ្ពស់ (Energy-efficient hardware) និងប្រព័ន្ធត្រជាក់កម្រិតខ្ពស់។
  • Software: កម្មវិធីសម្រាប់ Virtualization (ដូចជា VMware ឬ Hyper-V) និងកម្មវិធីគ្រប់គ្រងថាមពល (Power management software)។
  • Metrics Tools: ឧបករណ៍សម្រាប់វាស់វែងប្រសិទ្ធភាពដូចជា PUE (Power Usage Effectiveness) និង DCiE។

៣. ការពិនិត្យសម្រាប់បរិបទកម្ពុជា/អាស៊ីអាគ្នេយ៍

ភាពលំអៀងនៃទិន្នន័យ (Data Bias)៖

ការសិក្សានេះគឺជាការពិនិត្យឡើងវិញនូវឯកសារស្រាវជ្រាវសកល (Literature Review) ដោយអ្នកនិពន្ធមកពីប្រទេសនីហ្សេរីយ៉ា ដែលផ្តោតលើទិន្នន័យពីសហរដ្ឋអាមេរិក និងអឺរ៉ុប។ ទោះបីជាមិនមានទិន្នន័យជាក់លាក់ពីកម្ពុជាក៏ដោយ បញ្ហានៃការកើនឡើងថ្លៃថាមពល និងតម្រូវការមជ្ឈមណ្ឌលទិន្នន័យគឺជាបញ្ហាសកលដែលកម្ពុជាក៏កំពុងជួបប្រទះ។

លទ្ធភាពនៃការអនុវត្ត (Applicability)៖

គោលការណ៍នៃការសិក្សានេះមានប្រយោជន៍ខ្លាំងណាស់សម្រាប់កម្ពុជា ជាពិសេសដោយសារតែតម្លៃអគ្គិសនីនៅកម្ពុជាមានកម្រិតខ្ពស់ធៀបនឹងតំបន់។

  • វិស័យធនាគារ និងមីក្រូហិរញ្ញវត្ថុ: ធនាគារនៅកម្ពុជាដែលពង្រីកសេវា Digital Banking អាចប្រើប្រាស់ Virtualization ដើម្បីកាត់បន្ថយចំណាយលើ Data Center។
  • ក្រុមហ៊ុនផ្តល់សេវាអ៊ីនធឺណិត (ISPs): ការអនុវត្ត PUE ត្រឹមត្រូវអាចជួយ ISPs នៅភ្នំពេញកាត់បន្ថយការចំណាយប្រតិបត្តិការយ៉ាងច្រើន។
  • វិស័យអប់រំ និងសាកលវិទ្យាល័យ: សាកលវិទ្យាល័យអាចប្រើប្រាស់ Cloud Computing សម្រាប់ការស្រាវជ្រាវ និងការរក្សាទុកទិន្នន័យជំនួសឱ្យការទិញ Server ផ្ទាល់ខ្លួនច្រើន។

ការអនុវត្ត Green Computing មិនត្រឹមតែជួយបរិស្ថានប៉ុណ្ណោះទេ ប៉ុន្តែថែមទាំងជាយុទ្ធសាស្ត្រអាជីវកម្មដ៏សំខាន់សម្រាប់ស្ថាប័ននៅកម្ពុជាដើម្បីកាត់បន្ថយការចំណាយ។

៤. ផែនការសកម្មភាពសម្រាប់និស្សិត (Actionable Roadmap)

ដើម្បីអនុវត្តតាមការសិក្សានេះ និស្សិតគួរអនុវត្តតាមជំហានខាងក្រោម៖

  1. ជំហានទី ១៖ ការវាស់វែងប្រសិទ្ធភាពបច្ចុប្បន្ន: និស្សិតគួរចាប់ផ្តើមដោយសិក្សាពីរបៀបគណនា PUE (Power Usage Effectiveness) និង DCiE នៅក្នុងបន្ទប់ Server របស់សាលា ឬកន្លែងធ្វើការ ដើម្បីដឹងពីកម្រិតនៃការប្រើប្រាស់ថាមពលបច្ចុប្បន្ន។
  2. ជំហានទី ២៖ ការអនុវត្តបច្ចេកវិទ្យា Virtualization: រៀនប្រើប្រាស់កម្មវិធីដូចជា VMware ESXi ឬ Microsoft Hyper-V ដើម្បីបង្រួម Server ជាច្រើនឱ្យមកនៅក្នុងម៉ាស៊ីនតែមួយ ដែលជួយសន្សំសំចៃថាមពល និងទីកន្លែង។
  3. ជំហានទី ៣៖ ការកំណត់គោលនយោបាយគ្រប់គ្រងថាមពល: សិក្សាពីការប្រើប្រាស់ Power Management Features នៅក្នុងប្រព័ន្ធប្រតិបត្តិការ និងការកំណត់ម៉ាស៊ីនត្រជាក់ (Cooling optimization) ឱ្យសមស្របនឹងស្តង់ដារកុំព្យូទ័របៃតង។
  4. ជំហានទី ៤៖ ការពិចារណាលើ Cloud Services: វាយតម្លៃលើសេវាកម្ម Cloud ដូចជា AWS ឬ Google Cloud Platform សម្រាប់គម្រោងថ្មីៗ ជាជាងការទិញ Hardware បន្ថែម ដើម្បីកាត់បន្ថយការបង្កើតកាកសំណល់អេឡិចត្រូនិក (E-waste)។

៥. វាក្យសព្ទបច្ចេកទេស (Technical Glossary)

ពាក្យបច្ចេកទេស (English) ការពន្យល់ជាខេមរភាសា (Khmer Explanation) និយមន័យសាមញ្ញ (Simple Definition)
Virtualization បច្ចេកវិទ្យាដែលអនុញ្ញាតឱ្យបង្កើតម៉ាស៊ីននិម្មិត (Virtual Machines) ជាច្រើននៅលើកុំព្យូទ័រមេ (Physical Server) តែមួយ ដើម្បីប្រើប្រាស់ធនធាន Hardware ឱ្យអស់លទ្ធភាព និងកាត់បន្ថយចំនួនម៉ាស៊ីនដែលត្រូវទិញ។ ដូចជាការបែងចែកផ្ទះធំមួយឱ្យទៅជាបន្ទប់ជួលជាច្រើន ដើម្បីឱ្យមនុស្សច្រើនគ្រួសារអាចរស់នៅបាន ដោយមិនចាំបាច់សង់ផ្ទះថ្មីច្រើន។
Power Usage Effectiveness (PUE) ជាមាត្រដ្ឋានស្តង់ដារសម្រាប់វាស់វែងប្រសិទ្ធភាពនៃការប្រើប្រាស់ថាមពលនៅក្នុងមជ្ឈមណ្ឌលទិន្នន័យ ដោយប្រៀបធៀបថាមពលសរុបដែលចូលក្នុងអគារ ធៀបនឹងថាមពលដែលប្រើដោយឧបករណ៍ IT ជាក់ស្តែង។ ដូចជាការវាស់ថាតើទឹកប៉ុន្មានដែលហូរចូលស្រែជាក់ស្តែង ធៀបនឹងទឹកដែលបាត់បង់តាមផ្លូវដោយការហួតឬជ្រាបចូលដី។
Hypervisor ជាកម្មវិធីកុំព្យូទ័រ ឬ Firmware ដែលដើរតួជាអ្នកគ្រប់គ្រងនៅចន្លោះផ្នែករឹង (Hardware) និងប្រព័ន្ធប្រតិបត្តិការ (OS) ដើម្បីអនុញ្ញាតឱ្យ OS ដំណើរការច្រើនក្នុងពេលតែមួយលើម៉ាស៊ីនតែមួយ។ ដូចជាប៉ូលិសចរាចរណ៍ដែលគ្រប់គ្រងផ្លូវ ដើម្បីឱ្យឡានជាច្រើន (ប្រព័ន្ធប្រតិបត្តិការ) អាចធ្វើដំណើរលើផ្លូវតែមួយបានដោយសុវត្ថិភាព។
Cloud Computing ការផ្តល់សេវាកម្មកុំព្យូទ័រ (ដូចជា Server, Storage, Database) តាមរយៈអ៊ីនធឺណិត ដែលអនុញ្ញាតឱ្យអ្នកប្រើប្រាស់ជួលប្រើតាមតម្រូវការ ដោយមិនចាំបាច់ទិញម៉ាស៊ីនដាក់នៅកន្លែងផ្ទាល់ខ្លួន។ ដូចជាការប្រើប្រាស់អគ្គិសនីពីរដ្ឋ ដោយគ្រាន់តែបង់លុយតាមនាឡិកាជាជាងការទិញម៉ាស៊ីនភ្លើងមកដាក់នៅផ្ទះរៀងៗខ្លួន។
Blade Servers ជាប្រភេទកុំព្យូទ័រ Server ដែលមានរាងស្តើងដូចបន្ទះក្តារ ដែលត្រូវបានរចនាឡើងដើម្បីសន្សំសំចៃទីកន្លែង និងថាមពល ដោយដកចេញនូវសមាសធាតុមិនចាំបាច់មួយចំនួន ហើយដាក់បញ្ចូលគ្នាក្នុងប្រអប់រួមមួយ (Chassis)។ ដូចជាសៀវភៅដែលតម្រៀបគ្នានៅលើធ្នើរ គឺអាចដាក់បានច្រើនក្នុងទូតែមួយ ជាជាងការដាក់ប្រអប់ធំៗដាច់ដោយឡែក។
Sustainable IT Services ការផ្តល់សេវាកម្មព័ត៌មានវិទ្យាដែលមិនត្រឹមតែគិតពីប្រសិទ្ធភាពបច្ចេកទេស និងប្រាក់ចំណេញប៉ុណ្ណោះទេ ប៉ុន្តែថែមទាំងគិតគូរពីផលប៉ះពាល់បរិស្ថាន និងសង្គមក្នុងរយៈពេលវែងផងដែរ។ ដូចជាការធ្វើកសិកម្មដែលថែរក្សាដីឱ្យនៅមានជីជាតិសម្រាប់ឆ្នាំក្រោយៗ មិនមែនប្រើថ្នាំគីមីដើម្បីតែបានទិន្នផលច្រើននៅឆ្នាំនេះហើយដីខូចនៅឆ្នាំក្រោយនោះទេ។
