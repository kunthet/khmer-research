១. សេចក្តីសង្ខេបប្រតិបត្តិ (Executive Summary)

បញ្ហា (The Problem)៖ ការស្រាវជ្រាវនេះដោះស្រាយបញ្ហា "គំនិតរសាត់" (Concept Drift) នៅក្នុងប្រព័ន្ធអ៊ីនធឺណិតនៃអ្វីៗ (IoT) ដែលបណ្តាលឱ្យទិន្នន័យមានការផ្លាស់ប្តូរជានិច្ច និងធ្វើឱ្យប្រព័ន្ធរកឃើញការជ្រៀតចូល (IDS) បាត់បង់ប្រសិទ្ធភាពក្នុងការការពារការវាយប្រហារបណ្តាញ។

វិធីសាស្ត្រ (The Methodology)៖ ការសិក្សានេះប្រើប្រាស់វិធីសាស្ត្ររួមបញ្ចូលគ្នា (Ensemble learning approach) ដើម្បីកំណត់ទីតាំងនៃគំនិតរសាត់យ៉ាងជាក់លាក់ និងអនុវត្តម៉ូដែលសិក្សាតម្រូវតាមការផ្លាស់ប្តូរ (Adaptive learning model) ដើម្បីធ្វើបច្ចុប្បន្នភាពការការពារ។

  • EMNCD Algorithm (ក្បួនដោះស្រាយរួមបញ្ចូលគ្នានៃការធ្វើតេស្តស្ថិតិមិនប៉ារ៉ាម៉ែត្រ ដូចជា Kolmogorov-Smirnov, Wilcoxon rank sum, និង Mann-Kendall tests)","Isolation Forest (វិធីសាស្ត្រព្រៃដាច់ដោយឡែកសម្រាប់រកឃើញភាពមិនប្រក្រតីក្នុងទិន្នន័យអនឡាញ)","WOA-XGBoost (ការប្រើប្រាស់ Whale Optimization Algorithm ដើម្បីកែសម្រួលប៉ារ៉ាម៉ែត្រ XGBoost សម្រាប់សម្របតាមគំនិតរសាត់)","Edge-IIoTset Dataset (ការពិសោធន៍លើសំណុំទិន្នន័យសុវត្ថិភាព IoT ឧស្សាហកម្មជាក់ស្តែង និងសំណុំទិន្នន័យសិប្បនិម្មិត)

លទ្ធផលសំខាន់ៗ (The Verdict)៖

  • ក្បួនដោះស្រាយ EMNCD មានសមត្ថភាពកំណត់ចំណុចរសាត់ (Drift points) បានយ៉ាងត្រឹមត្រូវ និងមានប្រសិទ្ធភាពខ្ពស់ជាងវិធីសាស្ត្របុរាណលើសំណុំទិន្នន័យសិប្បនិម្មិត និងជាក់ស្តែង។","នៅពេលមានគំនិតរសាត់ ភាពត្រឹមត្រូវនៃម៉ូដែលដើមបានធ្លាក់ចុះមកត្រឹម ៥៦.៨០% ប៉ុន្តែម៉ូដែល WOA-XGBoost ដែលបានស្នើឡើងអាចស្តារភាពត្រឹមត្រូវជាមធ្យមបានរហូតដល់ ៩៣.៩៣%។","ការសិក្សានេះបង្ហាញថាការធ្វើបច្ចុប្បន្នភាពម៉ូដែលដោយស្វ័យប្រវត្តិអាចបង្កើនពិន្ទុ F1 ដល់ ៩៥.៦៦% ដែលធានាបាននូវសុវត្ថិភាពខ្ពស់ និងកាត់បន្ថយការជូនដំណឹងមិនពិតសម្រាប់ប្រព័ន្ធ IoT។

២. ការវិភាគលើប្រសិទ្ធភាព និងការចំណាយ (Performance & Constraints)

វិធីសាស្ត្រ (Method) គុណសម្បត្តិ (Pros) គុណវិបត្តិ (Cons) លទ្ធផលគន្លឹះ (Key Result)
EMNCD (Ensemble Multiple Non-parametric Concept Localization Detectors)
វិធីសាស្ត្រស្នើឡើងដែលរួមបញ្ចូលគ្នានូវការធ្វើតេស្តស្ថិតិមិនប៉ារ៉ាម៉ែត្រចំនួនបី (K-S, Wilcoxon, Mann-Kendall) ដើម្បីរកមើលគំនិតរសាត់		 មានភាពជាក់លាក់ខ្ពស់ក្នុងការកំណត់ទីតាំងរសាត់ (Drift Localization) និងកាត់បន្ថយអត្រានៃការជូនដំណឹងមិនពិត (False Positives) បានយ៉ាងល្អ។ តម្រូវឱ្យមានការគណនាស្ថិតិច្រើនជាងវិធីសាស្ត្រទោល ដោយសារត្រូវដំណើរការតេស្តបីក្នុងពេលតែមួយ។ អាចរកឃើញចំណុចរសាត់បានត្រឹមត្រូវ ១០០% លើសំណុំទិន្នន័យ SEA ដោយគ្មានការជូនដំណឹងមិនពិត និងមានការពន្យារពេល (Delay) ទាបជាងវិធីសាស្ត្រដទៃ។
Classical Methods (DDM, Page-Hinkley, HCDTs)
វិធីសាស្ត្របុរាណដែលពឹងផ្អែកលើអត្រាកំហុស (Error Rate) ឬការផ្លាស់ប្តូរស្ថិតិទោល		 មានភាពសាមញ្ញក្នុងការអនុវត្ត និងប្រើប្រាស់ធនធានគណនតិច។ មានអត្រាជូនដំណឹងមិនពិតខ្ពស់ និងបរាជ័យក្នុងការរកឃើញគំនិតរសាត់ក្នុងទិន្នន័យដែលមានភាពស្មុគស្មាញ ឬផ្លាស់ប្តូរបន្តិចម្តងៗ។ មានការជូនដំណឹងមិនពិត និងការខកខានក្នុងការរកឃើញ (Missed Detections) ច្រើននៅក្នុងសំណុំទិន្នន័យស្មុគស្មាញ។
WOA-XGBoost (Adaptive Model)
ម៉ូដែល XGBoost ដែលត្រូវបានកែសម្រួលដោយប្រើក្បួនដោះស្រាយ Whale Optimization Algorithm (WOA) បន្ទាប់ពីរកឃើញគំនិតរសាត់		 អាចស្តារសមត្ថភាពចំណាត់ថ្នាក់ (Classification) ឱ្យមកនៅកម្រិតខ្ពស់វិញបានយ៉ាងឆាប់រហ័សបន្ទាប់ពីមានការផ្លាស់ប្តូរទិន្នន័យ។ ទាមទារពេលវេលាសម្រាប់ការបង្វឹកម៉ូដែលឡើងវិញ (Retraining) និងការស្វែងរកប៉ារ៉ាម៉ែត្រថ្មីដោយ WOA។ បង្កើនភាពត្រឹមត្រូវពី ៥៦.៨០% (បើមិនមានការសម្របខ្លួន) ទៅ ៩៣.៩៣% (ជាមួយការសម្របខ្លួន) លើសំណុំទិន្នន័យ Edge-IIoTset ។

ការចំណាយលើធនធាន (Resource Cost)៖ ការសិក្សានេះប្រើប្រាស់វិធីសាស្ត្រដែលត្រូវបានរចនាឡើងសម្រាប់ប្រព័ន្ធ IoT ដូច្នេះធនធានដែលត្រូវការគឺស្ថិតក្នុងកម្រិតមធ្យម ប៉ុន្តែការបង្វឹកម៉ូដែលឡើងវិញអាចទាមទារធនធានបន្ថែម។

  • Computing Hardware: អាចដំណើរការលើកុំព្យូទ័រដែលមាន CPU ធម្មតាសម្រាប់ការរកឃើញ ប៉ុន្តែការបង្វឹកម៉ូដែល WOA-XGBoost អាចត្រូវការធនធានខ្លាំងជាងនេះបន្តិច ឬ GPU ដើម្បីល្បឿនលឿន។
  • Software & Libraries: Python (Scikit-learn, XGBoost) សម្រាប់ការអនុវត្តម៉ូដែល និងការវិភាគទិន្នន័យ។
  • Dataset: Edge-IIoTset (ទិន្នន័យចរាចរណ៍បណ្តាញ IoT ឧស្សាហកម្ម) ដែលមានទំហំធំល្មមសម្រាប់ការពិសោធន៍។

ភាពលំអៀងនៃទិន្នន័យ (Data Bias)៖

ការសិក្សានេះប្រើប្រាស់សំណុំទិន្នន័យ 'Edge-IIoTset' ដែលជាទិន្នន័យសុវត្ថិភាពតាមបច្ចេកវិទ្យា Edge Computing និង IIoT ជាក់ស្តែង រួមជាមួយទិន្នន័យសិប្បនិម្មិត (SEA, Stagger)។ ទោះបីជាមិនមែនជាទិន្នន័យពីកម្ពុជាដោយផ្ទាល់ ប៉ុន្តែវាឆ្លុះបញ្ចាំងពីការវាយប្រហារតាមអ៊ីនធឺណិតស្តង់ដារសកល (ដូចជា DDoS, SQL Injection) ដែលពាក់ព័ន្ធយ៉ាងខ្លាំងចំពោះហេដ្ឋារចនាសម្ព័ន្ធឌីជីថលនៅកម្ពុជា។

លទ្ធភាពនៃការអនុវត្ត (Applicability)៖

វិធីសាស្ត្រនេះមានប្រយោជន៍ខ្លាំងណាស់សម្រាប់កម្ពុជា ជាពិសេសក្នុងបរិបទនៃការរីកចម្រើនវិស័យឌីជីថល និងឧស្សាហកម្ម ៤.០។

  • តំបន់សេដ្ឋកិច្ចពិសេស (SEZs): រោងចក្រទំនើបៗក្នុងតំបន់សេដ្ឋកិច្ចពិសេសនៅភ្នំពេញ ឬក្រុងព្រះសីហនុ ដែលប្រើប្រាស់ឧបករណ៍ IoT អាចប្រើប្រព័ន្ធនេះដើម្បីការពារការវាយប្រហារបណ្តាញ។
  • អ្នកផ្តល់សេវាអ៊ីនធឺណិត (ISPs): ក្រុមហ៊ុនអ៊ីនធឺណិតនៅកម្ពុជាអាចប្រើបច្ចេកវិទ្យានេះដើម្បីតាមដានចរាចរណ៍បណ្តាញដែលមានការប្រែប្រួលខ្ពស់ និងទប់ស្កាត់ការវាយប្រហារ។
  • គម្រោងទីក្រុងឆ្លាតវៃ (Smart Cities): ប្រព័ន្ធគ្រប់គ្រងចរាចរណ៍ ឬកាមេរ៉ាសុវត្ថិភាពសាធារណៈដែលតភ្ជាប់បណ្តាញ អាចការពារពីការជ្រៀតចូលដោយប្រើវិធីសាស្ត្ររកឃើញគំនិតរសាត់នេះ។

ការអនុវត្ត EMNCD និង WOA-XGBoost នឹងជួយពង្រឹងសន្តិសុខតាមអ៊ីនធឺណិតសម្រាប់ហេដ្ឋារចនាសម្ព័ន្ធសំខាន់ៗនៅកម្ពុជា ដោយកាត់បន្ថយហានិភ័យពីការវាយប្រហារដែលមិនធ្លាប់ស្គាល់។

៤. ផែនការសកម្មភាពសម្រាប់និស្សិត (Actionable Roadmap)

ដើម្បីអនុវត្តតាមការសិក្សានេះ និស្សិតគួរអនុវត្តតាមជំហានខាងក្រោម៖

  1. ១. ការសិក្សាមូលដ្ឋានគ្រឹះ: និស្សិតគួរចាប់ផ្តើមដោយសិក្សាអំពីស្ថិតិមិនប៉ារ៉ាម៉ែត្រ (Non-parametric statistics) ដូចជា Kolmogorov-Smirnov test និងគំនិតនៃ Concept Drift នៅក្នុង Machine Learning។
  2. ២. ការរៀបចំទិន្នន័យ: ទាញយកសំណុំទិន្នន័យ Edge-IIoTset និងអនុវត្តការសម្អាតទិន្នន័យ (Data Preprocessing) ដោយប្រើបណ្ណាល័យ Python Pandas ដើម្បីយល់ពីប្រភេទនៃការវាយប្រហារផ្សេងៗ។
  3. ៣. ការអនុវត្តម៉ូដែល EMNCD: សរសេរកូដដើម្បីបង្កើតប្រព័ន្ធរាវរកគំនិតរសាត់ដោយប្រើ Python SciPy សម្រាប់តេស្តស្ថិតិ និងប្រើប្រាស់បច្ចេកទេស Sliding Window ដើម្បីវិភាគទិន្នន័យជាលំហូរ។
  4. ៤. ការធ្វើសមាហរណកម្មជាមួយ XGBoost: ប្រើប្រាស់ XGBoost ជាអ្នកធ្វើចំណាត់ថ្នាក់ (Classifier) និងអនុវត្ត Whale Optimization Algorithm (WOA) ដើម្បីស្វែងរកប៉ារ៉ាម៉ែត្រដែលល្អបំផុតនៅពេលរកឃើញការផ្លាស់ប្តូរទិន្នន័យ។
  5. ៥. ការវាយតម្លៃប្រសិទ្ធភាព: ធ្វើការប្រៀបធៀបលទ្ធផលរវាងម៉ូដែលដែលមិនមានការសម្របខ្លួន និងម៉ូដែលដែលបានស្នើឡើង ដោយផ្តោតលើពិន្ទុ F1-score និង Accuracy ដើម្បីវាស់វែងភាពប្រសើរឡើង។

៥. វាក្យសព្ទបច្ចេកទេស (Technical Glossary)

ពាក្យបច្ចេកទេស (English) ការពន្យល់ជាខេមរភាសា (Khmer Explanation) និយមន័យសាមញ្ញ (Simple Definition)
Concept drift គឺជាបាតុភូតនៅក្នុងវិទ្យាសាស្ត្រទិន្នន័យដែលលក្ខណៈសម្បត្តិនៃទិន្នន័យគោលដៅមានការផ្លាស់ប្តូរតាមពេលវេលា ដែលបណ្តាលឱ្យម៉ូដែល AI ដែលធ្លាប់តែមានភាពត្រឹមត្រូវពីមុន ក្លាយជាលែងមានប្រសិទ្ធភាព ឬធ្វើការទស្សន៍ទាយខុស។ ដូចជាសិស្សម្នាក់ដែលរៀនត្រៀមប្រឡងដោយប្រើសៀវភៅចាស់ឆ្នាំមុន ប៉ុន្តែវិញ្ញាសាប្រឡងឆ្នាំនេះត្រូវបានគេផ្លាស់ប្តូរថ្មីទាំងស្រុង ធ្វើឱ្យចំណេះដឹងចាស់លែងត្រូវនឹងស្ថានការណ៍ថ្មី។
Non-parametric statistical methods គឺជាវិធីសាស្ត្រវិភាគស្ថិតិដែលមិនទាមទារឱ្យទិន្នន័យមានទម្រង់ចែកចាយជាក់លាក់ (ដូចជា Normal Distribution) ឡើយ។ នៅក្នុងការសិក្សានេះ វាត្រូវបានប្រើដើម្បីប្រៀបធៀបទិន្នន័យចាស់និងថ្មី ដើម្បីរកមើលការផ្លាស់ប្តូរ ដោយមិនខ្វល់ថាទិន្នន័យមានទម្រង់រញ៉េរញ៉ៃយ៉ាងណានោះទេ។ ដូចជាការប្រើឧបករណ៍ដែលអាចវាស់កម្ពស់មនុស្សបានគ្រប់គ្នា មិនថាពួកគេឈរត្រង់ ឬឈរកោងខ្នង ដោយមិនចាំបាច់តម្រូវឱ្យពួកគេឈរក្នុងទម្រង់ស្តង់ដារតែមួយ។
Isolation Forest (iForest) ជាក្បួនដោះស្រាយសម្រាប់ស្វែងរកភាពមិនប្រក្រតី (Anomaly Detection) ដោយព្យាយាមបំបែកទិន្នន័យនីមួយៗចេញពីគ្នា។ ទិន្នន័យដែលខុសគេខ្លាំង (Outlier) ងាយនឹងត្រូវបានបំបែកចេញលឿនជាងទិន្នន័យធម្មតា ដែលធ្វើឱ្យវារកឃើញការវាយប្រហារបានលឿន។ ដូចជាការស្វែងរកគ្រាប់ក្រួសនៅក្នុងបាវអង្ករ; គ្រាប់ក្រួសងាយនឹងត្រូវបានរើសចេញមុនគេនិងងាយសម្គាល់ ព្រោះវាមានលក្ខណៈខុសប្លែកពីគ្រាប់អង្ករភាគច្រើន។
Whale Optimization Algorithm (WOA) ជាក្បួនដោះស្រាយសម្រាប់ស្វែងរកដំណោះស្រាយល្អបំផុត (Optimization) ដោយយកគំរូតាមឥរិយាបថបរបាញ់របស់ត្រីបាឡែន (Bubble-net feeding) ដើម្បីស្វែងរកការកំណត់ប៉ារ៉ាម៉ែត្រដែលធ្វើឱ្យម៉ូដែលដំណើរការបានល្អបំផុត។ ដូចជាអ្នកនេសាទដែលរៀនបច្ចេកទេសបាចសំណាញ់ពិសេសដើម្បីចាប់ត្រីឱ្យបានច្រើនបំផុត ដោយសង្កេតមើលនិងរៀនពីចលនារបស់ត្រី។
Ensemble learning គឺជាបច្ចេកទេសនៃការប្រើប្រាស់ម៉ូដែល ឬវិធីសាស្ត្រវិភាគច្រើនបញ្ចូលគ្នា ដើម្បីធ្វើការសម្រេចចិត្តរួមគ្នា។ នៅក្នុងឯកសារនេះ EMNCD ប្រើវិធីសាស្ត្រស្ថិតិ ៣ ផ្សេងគ្នាបញ្ចូលគ្នា ដើម្បីធានាថាការរកឃើញគំនិតរសាត់មានភាពត្រឹមត្រូវខ្ពស់។ ដូចជាការសួរយោបល់ពីគ្រូពេទ្យជំនាញ ៣ នាក់ផ្សេងគ្នា ដើម្បីធានាថាការព្យាបាលមួយត្រឹមត្រូវ ជាជាងជឿលើការវិនិច្ឆ័យរបស់គ្រូពេទ្យតែម្នាក់។
