អ្នកនិពន្ធ៖ N. S. Koti Mani Kumar Tirumanadham (Bharath Institute of Higher Education and Research), S. Thaiyalnayaki (Bharath Institute of Higher Education and Research), V. Ganesan (Bharath Institute of Higher Education and Research)

១. សេចក្តីសង្ខេបប្រតិបត្តិ (Executive Summary)

បញ្ហា (The Problem)៖ ការស្រាវជ្រាវនេះដោះស្រាយបញ្ហាប្រឈមនៅក្នុងប្រព័ន្ធសិក្សាតាមអេឡិចត្រូនិក (E-Learning) ដែលទិន្នន័យមានភាពមិនស្មើគ្នា (Class Imbalance) និងមានទិន្នន័យមិនប្រក្រតី (Outliers) ដែលធ្វើឱ្យប៉ះពាល់ដល់សមត្ថភាពទស្សន៍ទាយលទ្ធផលសិក្សារបស់សិស្ស និងការផ្តល់មតិកែលម្អទាន់ពេលវេលា។

វិធីសាស្ត្រ (The Methodology)៖ អ្នកស្រាវជ្រាវបានស្នើឡើងនូវក្របខណ្ឌបច្ចេកវិទ្យាដែលប្រើប្រាស់ការរៀនរបស់ម៉ាស៊ីន (Machine Learning) រួមជាមួយការសម្អាតទិន្នន័យកម្រិតខ្ពស់ ដើម្បីវិភាគលើសកម្មភាពសិស្ស និងកសាងម៉ូដែលទស្សន៍ទាយដែលមានប្រសិទ្ធភាព។

  • ការប្រើប្រាស់បច្ចេកទេស Z-score ដើម្បីកំណត់និងដោះស្រាយទិន្នន័យមិនប្រក្រតី (Outliers) និង Min-Max Scaling សម្រាប់ធ្វើឱ្យទិន្នន័យមានស្តង់ដារ។
  • ការជ្រើសរើសលក្ខណៈពិសេសនៃទិន្នន័យដោយប្រើវិធីសាស្ត្រ Ridge-RFE (Ridge Regression and Recursive Feature Elimination) ដើម្បីកាត់បន្ថយភាពស្មុគស្មាញនៃម៉ូដែល។
  • ការកសាងម៉ូដែលទស្សន៍ទាយដោយប្រើក្បួនដោះស្រាយ Gradient Boosting Machine (GBM) លើទិន្នន័យដែលទទួលបានពី Kaggle។

លទ្ធផលសំខាន់ៗ (The Verdict)៖

  • ការអនុវត្តម៉ូដែល Gradient Boosting Machine ទទួលបានភាពត្រឹមត្រូវរហូតដល់ ៩៤% (94% accuracy) ក្នុងការទស្សន៍ទាយលទ្ធផលអ្នកសិក្សា ដែលខ្ពស់ជាងវិធីសាស្ត្រដទៃទៀត។
  • វិធីសាស្ត្រ Ridge-RFE បានរកឃើញថា "ឆមាស" (semester) និង "ការមើលសេចក្តីជូនដំណឹង" (announcements_view) គឺជាកត្តាសំខាន់បំផុតសម្រាប់ការព្យាករណ៍លទ្ធផល។
  • លទ្ធផលបង្ហាញថាការរួមបញ្ចូលគ្នានៃការវិភាគទិន្នន័យជាក់ស្តែង (Real-time analytics) ជាមួយ ML អាចជួយបង្កើតបរិយាកាសសិក្សាដែលឆ្លើយតបទៅនឹងតម្រូវការបុគ្គលបានប្រសើរជាងមុន និងផ្តល់អន្តរាគមន៍ទាន់ពេលវេលា។

២. ការវិភាគលើប្រសិទ្ធភាព និងការចំណាយ (Performance & Constraints)

វិធីសាស្ត្រ (Method) គុណសម្បត្តិ (Pros) គុណវិបត្តិ (Cons) លទ្ធផលគន្លឹះ (Key Result)
Ridge-RFE + GBM (Proposed Method)
ការជ្រើសរើសលក្ខណៈពិសេសតាមបែប Ridge-RFE បូកនឹងម៉ូដែល Gradient Boosting Machine		 មានសមត្ថភាពខ្ពស់ក្នុងការជ្រើសរើសកត្តាសំខាន់ៗ (Feature Selection) និងកាត់បន្ថយភាពស្មុគស្មាញនៃម៉ូដែល ព្រមទាំងមានប្រសិទ្ធភាពខ្ពស់លើទិន្នន័យដែលមានរចនាសម្ព័ន្ធ (Structured Data)។ ទាមទារធនធានកុំព្យូទ័រខ្លាំងសម្រាប់ដំណើរការទិន្នន័យតាមពេលវេលាជាក់ស្តែង (Real-time analytics) និងការបណ្តុះបណ្តាល។ សម្រេចបានភាពត្រឹមត្រូវ (Accuracy) ៩៤%
Multimodal Data (Gupta et al., 2024)
ការប្រើប្រាស់ទិន្នន័យចម្រុះ (ដូចជាការតាមដានគ្រាប់ភ្នែក និងទឹកមុខ)		 អាចវិភាគស៊ីជម្រៅទៅលើស្ថានភាពការយល់ដឹង (Cognitive State) របស់សិស្សបានយ៉ាងល្អ។ ត្រូវការឧបករណ៍ចាប់សញ្ញា (Sensors) និងហេដ្ឋារចនាសម្ព័ន្ធស្មុគស្មាញ ដែលពិបាកអនុវត្តក្នុងបរិបទសាលារៀនទូទៅ។ សម្រេចបានភាពត្រឹមត្រូវ ៩២.៥៨%
Oversampling Techniques (Tariq et al., 2023)
បច្ចេកទេសបង្កើនចំនួនទិន្នន័យដូចជា SMOTE ឬ ADASYN		 ដោះស្រាយបញ្ហាអតុល្យភាពទិន្នន័យ (Class Imbalance) បានល្អសម្រាប់ទិន្នន័យអប់រំ។ មានកម្រិតភាពត្រឹមទាបជាងបើធៀបនឹងវិធីសាស្ត្រ Hybrid ដែលស្នើឡើងក្នុងអត្ថបទនេះ។ សម្រេចបានភាពត្រឹមត្រូវ ៨៣.៧%

ការចំណាយលើធនធាន (Resource Cost)៖ ការអនុវត្តវិធីសាស្ត្រនេះទាមទារឱ្យមានហេដ្ឋារចនាសម្ព័ន្ធកុំព្យូទ័រដែលមានសមត្ថភាពខ្ពស់ ដើម្បីគាំទ្រការវិភាគទិន្នន័យតាមពេលវេលាជាក់ស្តែង (Real-time analytics)។

  • Hardware: ម៉ាស៊ីនមេ (Server) ឬកុំព្យូទ័រដែលមានសមត្ថភាពខ្ពស់ (High-performance computing) សម្រាប់ដំណើរការម៉ូដែល GBM លើទិន្នន័យជាច្រើន។
  • Dataset: ទិន្នន័យពី Kaggle ដែលមានផ្ទុកព័ត៌មានអំពីសកម្មភាពសិស្ស (Student Engagement Metrics) និងប្រវត្តិរូប។
  • Software & Libraries: ភាសា Python និងបណ្ណាល័យ Machine Learning ដូចជា Scikit-learn សម្រាប់ Ridge-RFE និងឧបករណ៍សម្រាប់ GBM។

ភាពលំអៀងនៃទិន្នន័យ (Data Bias)៖

ការសិក្សានេះប្រើប្រាស់ទិន្នន័យពី Kaggle ដែលទិន្នន័យដើមទំនងជាបានមកពីបរិបទអប់រំនៅមជ្ឈិមបូព៌ា (ផ្អែកលើលក្ខណៈពិសេសដូចជា 'Nationality' និង 'Place of Birth')។ នេះអាចបង្កើតជាភាពលំអៀង (Bias) នៅពេលយកមកអនុវត្តនៅកម្ពុជា ដោយសារឥរិយាបថសិស្ស និងលទ្ធភាពប្រើប្រាស់បច្ចេកវិទ្យាអាចមានភាពខុសគ្នា។

លទ្ធភាពនៃការអនុវត្ត (Applicability)៖

វិធីសាស្ត្រនេះមានសារៈសំខាន់ខ្លាំងសម្រាប់កម្ពុជា ស្របពេលដែលក្រសួង និងគ្រឹះស្ថានសិក្សាកំពុងជំរុញការរៀនតាមប្រព័ន្ធឌីជីថល (E-Learning)។

  • សាកលវិទ្យាល័យធំៗ (RUPP, ITC): អាចយកទៅអនុវត្តក្នុងប្រព័ន្ធគ្រប់គ្រងការសិក្សា (LMS) ដូចជា Moodle ដើម្បីទស្សន៍ទាយសិស្សដែលអាចនឹងបោះបង់ការសិក្សា ឬរៀនខ្សោយ។
  • ក្រសួងអប់រំ យុវជន និងកីឡា (MoEYS): អាចប្រើប្រាស់សម្រាប់ការវិភាគទិន្នន័យលើថ្នាលឌីជីថលជាតិ ដើម្បីកែលម្អមាតិកា និងផ្តល់ការគាំទ្រដល់សិស្សនៅតាមតំបន់ដាច់ស្រយាល។

ទោះបីជាមានសក្តានុពលខ្ពស់ ប៉ុន្តែការអនុវត្តជាក់ស្តែងនៅកម្ពុជាទាមទារឱ្យមានការប្រមូលទិន្នន័យក្នុងស្រុកជាមុនសិន ដើម្បីធានាថាម៉ូដែល (Model) ឆ្លុះបញ្ចាំងពីតថភាពជាក់ស្តែងរបស់សិស្សខ្មែរ។

  1. ការសិក្សាមូលដ្ឋានគ្រឹះ និងប្រមូលទិន្នន័យ: ចាប់ផ្តើមដោយការសិក្សាអំពី Python libraries ដូចជា Scikit-learn និងទាញយកទិន្នន័យពី Kaggle ដើម្បីយល់អំពីលក្ខណៈពិសេស (Features) ដូចជាវត្តមាន និងការចូលមើលមាតិកា។
  2. ការរៀបចំទិន្នន័យ (Data Preprocessing): អនុវត្តបច្ចេកទេស Z-score សម្រាប់សម្អាតទិន្នន័យមិនប្រក្រតី និង Min-Max Scaling ដើម្បីធ្វើឱ្យទិន្នន័យមានស្តង់ដារដូចការណែនាំក្នុងអត្ថបទ។
  3. ការជ្រើសរើសលក្ខណៈពិសេស និងបណ្តុះបណ្តាលម៉ូដែល: ប្រើប្រាស់ Ridge Regression ជាមួយ Recursive Feature Elimination (RFE) ដើម្បីកំណត់កត្តាសំខាន់ៗ រួចបណ្តុះបណ្តាលម៉ូដែលដោយប្រើ Gradient Boosting Machine (GBM)។
  4. ការធ្វើតេស្តសាកល្បងក្នុងបរិបទកម្ពុជា: សហការជាមួយសាកលវិទ្យាល័យក្នុងស្រុក ដើម្បីសាកល្បងប្រមូលទិន្នន័យតូចមួយ (Pilot Data) ពីប្រព័ន្ធ LMS របស់ពួកគេ និងផ្ទៀងផ្ទាត់ប្រសិទ្ធភាពម៉ូដែល។

៥. វាក្យសព្ទបច្ចេកទេស (Technical Glossary)

ពាក្យបច្ចេកទេស (English) ការពន្យល់ជាខេមរភាសា (Khmer Explanation) និយមន័យសាមញ្ញ (Simple Definition)
Gradient Boosting Machine (GBM) គឺជាបច្ចេកទេសម៉ាស៊ីនរៀន (Machine Learning) ដែលបង្កើតម៉ូដែលទស្សន៍ទាយដ៏ខ្លាំងមួយ ដោយការបូកបញ្ចូលគ្នានូវម៉ូដែលតូចៗជាច្រើនជាបន្តបន្ទាប់។ ម៉ូដែលថ្មីនីមួយៗត្រូវបានបង្កើតឡើងដើម្បីកែតម្រូវកំហុសដែលម៉ូដែលមុនបានធ្វើខុស ដែលធ្វើឱ្យលទ្ធផលចុងក្រោយមានភាពត្រឹមត្រូវខ្ពស់។ ដូចជាការធ្វើការងារជាក្រុម ដែលសមាជិកម្នាក់ៗចូលមកជួយកែចំណុចខ្វះខាតរបស់សមាជិកមុន ដើម្បីឱ្យការងារសម្រេចបានល្អឥតខ្ចោះ។
Ridge-RFE ជាវិធីសាស្ត្រជ្រើសរើសទិន្នន័យសំខាន់ៗ (Feature Selection) ដោយរួមបញ្ចូលគ្នានូវ Ridge Regression (ដើម្បីកាត់បន្ថយបញ្ហាទិន្នន័យស្ទួនគ្នា) និង Recursive Feature Elimination (ដើម្បីដកទិន្នន័យដែលមិនសំខាន់ចេញម្តងមួយៗ)។ វាជួយឱ្យម៉ាស៊ីនរៀនផ្តោតតែលើកត្តាដែលមានឥទ្ធិពលបំផុតលើលទ្ធផល។ ប្រៀបដូចជាការរៀបចំវ៉ាលីធ្វើដំណើរ ដោយយើងពិនិត្យមើលរបស់ម្តងមួយៗ ហើយដករបស់ណាដែលមិនចាំបាច់ចេញ រហូតដល់សល់តែរបស់សំខាន់បំផុត។
Class Imbalance ស្ថានភាពដែលនៅក្នុងទិន្នន័យ មានក្រុមមួយមានចំនួនច្រើនលើសលប់ជាងក្រុមមួយទៀត (ឧទាហរណ៍៖ មានសិស្សជាប់ ៩០% និងសិស្សធ្លាក់ ១០%)។ នេះធ្វើឱ្យម៉ាស៊ីនរៀនមានភាពលំអៀង ដោយវានឹងទស្សន៍ទាយថា 'ជាប់' ជានិច្ច ព្រោះវាឃើញទិន្នន័យនោះច្រើនជាង។ ដូចជាការរៀនសម្គាល់ផ្លែឈើ បើគ្រូបង្រៀនបង្ហាញតែរូបផ្លែប៉ោម ១០០ រូប និងផ្លែក្រូចតែ ១ រូប សិស្សនឹងស្គាល់តែផ្លែប៉ោម ហើយពិបាកសម្គាល់ផ្លែក្រូច។
Z-score ជាវិធីសាស្ត្រស្ថិតិសម្រាប់វាស់វែងថាតើទិន្នន័យមួយស្ថិតនៅឆ្ងាយប៉ុណ្ណាពីមធ្យមភាគ (Mean)។ នៅក្នុងការសិក្សានេះ វាត្រូវបានប្រើដើម្បីរកមើលទិន្នន័យមិនប្រក្រតី (Outliers) ដូចជាពិន្ទុដែលខ្ពស់ពេក ឬទាបពេកខុសពីធម្មតា ដើម្បីកុំឱ្យវាប៉ះពាល់ដល់ការវិភាគ។ ដូចជាការកំណត់ថាសិស្សណាមានកម្ពស់ខុសប្លែកខ្លាំងពីមិត្តរួមថ្នាក់ (ខ្ពស់កប់ពពក ឬទាបខ្លាំង) ដោយប្រៀបធៀបជាមួយកម្ពស់មធ្យមរបស់សិស្សទាំងថ្នាក់។
Min-Max Scaling ដំណើរការប្លែងទិន្នន័យដែលមានខ្នាតខុសៗគ្នា (ឧទាហរណ៍៖ អាយុជាឆ្នាំ និងពិន្ទុជាលេខ) ឱ្យទៅជាតម្លៃស្តង់ដារនៅចន្លោះពី ០ ទៅ ១។ នេះជួយឱ្យម៉ាស៊ីនរៀនមិនផ្តល់អាទិភាពលើទិន្នន័យដែលមានលេខធំជាងទិន្នន័យដែលមានលេខតូច។ ប្រៀបដូចជាការប្តូរពិន្ទុពីមុខវិជ្ជាផ្សេងៗដែលមានពិន្ទុពេញខុសគ្នា (ខ្លះ ១០០ ខ្លះ ៥០) មកជាភាគរយ (%) ដូចគ្នា ដើម្បីងាយស្រួលប្រៀបធៀប។
Multicollinearity បាតុភូតដែលទិន្នន័យ (Variables) ពីរឬច្រើននៅក្នុងគំរូមានទំនាក់ទំនងគ្នាខ្លាំងពេក (ឧទាហរណ៍៖ 'អាយុ' និង 'ឆ្នាំកំណើត')។ នេះធ្វើឱ្យម៉ាស៊ីនរៀនពិបាកបែងចែកថាតើមួយណាជាមូលហេតុពិតប្រាកដនៃលទ្ធផល។ Ridge Regression ត្រូវបានប្រើដើម្បីដោះស្រាយបញ្ហានេះ។ ដូចជាមានមនុស្សពីរនាក់និយាយប្រាប់រឿងដូចគ្នាព្រមគ្នា ធ្វើឱ្យយើងពិបាកដឹងថាអ្នកណាជាអ្នកនិយាយការពិត ឬអ្នកណាមានឥទ្ធិពលជាង។

៦. ប្រធានបទពាក់ព័ន្ធ (Further Reading)

ប្រធានបទ និងសំណួរស្រាវជ្រាវដែលទាក់ទងនឹងឯកសារនេះ ដែលអ្នកអាចស្វែងរកបន្ថែម៖

  • handling class imbalance in educational data mining datasets
  • gradient boosting machine for student performance prediction
  • multimodal learning analytics in e-learning environments
  • recursive feature elimination with ridge regression for feature selection
