១. សេចក្តីសង្ខេបប្រតិបត្តិ (Executive Summary)

បញ្ហា (The Problem)៖ អក្សរសិល្ប៍បច្ចុប្បន្នទាក់ទងនឹងកសិកម្មឆ្លាតវៃ (Smart Farming) នៅខ្វះមូលដ្ឋានគ្រឹះនៃវិធីសាស្ត្រ ឬគំរូស្ថាបត្យកម្មដែលពេញលេញ ដោយភាគច្រើនផ្តោតតែលើករណីប្រើប្រាស់ជាក់លាក់ ឬបច្ចេកវិទ្យាតែមួយផ្នែកប៉ុណ្ណោះ។

វិធីសាស្ត្រ (The Methodology)៖ អ្នកនិពន្ធបានអនុវត្តការសិក្សាស្រាវជ្រាវឯកសារជាប្រព័ន្ធ ដើម្បីវិភាគនិងប្រមូលផ្តុំវិធីសាស្ត្រស្ថាបត្យកម្មដែលមានស្រាប់ឱ្យទៅជាគំរូស្តង់ដារមួយ។

  • Systematic Literature Review (ការត្រួតពិនិត្យឯកសារជាប្រព័ន្ធ) ដោយផ្អែកលើវិធីសាស្ត្ររបស់ Webster និង Watson លើឯកសារចំនួន ១៨ ។
  • Concept Matrix Aggregation (ការប្រមូលផ្តុំតាមរយៈម៉ាទ្រីសគំនិត) ដើម្បីកំណត់សមាសធាតុសំខាន់ៗនៃស្ថាបត្យកម្ម។

លទ្ធផលសំខាន់ៗ (The Verdict)៖

  • ការសិក្សានេះបានបង្កើតស្ថាបត្យកម្មប្រព័ន្ធព័ត៌មានទូទៅមួយសម្រាប់កសិកម្មឆ្លាតវៃ ដែលមានសមាសធាតុស្នូលចំនួន ៤ គឺ៖ ផលិតផលឆ្លាតវៃ (Smart Product), ការតភ្ជាប់បណ្តាញ (Network Connection), ទិន្នន័យ (Data), និងសេវាកម្មឆ្លាតវៃ (Smart Service)។
  • ស្ថាបត្យកម្មនេះបង្ហាញពីការតភ្ជាប់រវាងស្រទាប់ឧបករណ៍រូបវន្ត (Physical Layer) ដូចជាឧបករណ៍ចាប់សញ្ញា និងម៉ាស៊ីន ទៅកាន់ប្រព័ន្ធគ្រប់គ្រងទិន្នន័យ និងសេវាកម្មវិភាគ ដើម្បីគាំទ្រដល់ការសម្រេចចិត្តរបស់កសិករ។
  • លទ្ធផលនៃការស្រាវជ្រាវនេះបម្រើជាប្លង់មេ (Blueprint) សម្រាប់ការអភិវឌ្ឍប្រព័ន្ធថ្មី និងជាឧបករណ៍វិភាគសម្រាប់វាយតម្លៃប្រព័ន្ធកសិកម្មឆ្លាតវៃដែលមានស្រាប់ ដោយមិនពឹងផ្អែកលើបច្ចេកវិទ្យាជាក់លាក់ណាមួយឡើយ។

២. ការវិភាគលើប្រសិទ្ធភាព និងការចំណាយ (Performance & Constraints)

វិធីសាស្ត្រ (Method) គុណសម្បត្តិ (Pros) គុណវិបត្តិ (Cons) លទ្ធផលគន្លឹះ (Key Result)
Existing Specific Architectures (Ad-hoc Approaches)
ស្ថាបត្យកម្មជាក់លាក់ដែលមានស្រាប់ (វិធីសាស្ត្រ Ad-hoc)		 មានប្រសិទ្ធភាពសម្រាប់ករណីប្រើប្រាស់តែមួយជាក់លាក់ (ឧទាហរណ៍៖ ការដាំប៉េងប៉ោះ ឬការចិញ្ចឹមជ្រូក) និងងាយស្រួលអនុវត្តក្នុងកម្រិតតូច។ ខ្វះមូលដ្ឋានគ្រឹះនៃវិធីសាស្ត្ររឹងមាំ ពិបាកក្នុងការប្រើប្រាស់ឡើងវិញសម្រាប់វិស័យផ្សេង ហើយផ្តោតខ្លាំងពេកលើបច្ចេកវិទ្យាជាជាងប្រព័ន្ធទាំងមូល។ ដំណោះស្រាយដាច់ដោយឡែកដែលមិនអាចធ្វើមាត្រដ្ឋាន (Scalability) បានល្អ។
Proposed Generic IS Architecture
ស្ថាបត្យកម្មប្រព័ន្ធព័ត៌មានទូទៅដែលបានស្នើឡើង		 មានលក្ខណៈទូទៅដែលអាចអនុវត្តបានគ្រប់វិស័យកសិកម្ម មិនអាស្រ័យលើបច្ចេកវិទ្យាណាមួយ និងគ្របដណ្តប់ទាំង ៤ ស្រទាប់សំខាន់ៗ (Product, Network, Data, Service)។ នៅជាគំរូទ្រឹស្តី (Conceptual Model) ដែលត្រូវការការកែសម្រួលបន្ថែមនៅពេលយកទៅអនុវត្តជាក់ស្តែងក្នុងបរិបទនីមួយៗ។ បង្កើតបានជាប្លង់មេ (Blueprint) ស្តង់ដារសម្រាប់ការអភិវឌ្ឍប្រព័ន្ធកសិកម្មឆ្លាតវៃ។

ការចំណាយលើធនធាន (Resource Cost)៖ ការអនុវត្តស្ថាបត្យកម្មនេះតម្រូវឱ្យមានការវិនិយោគលើហេដ្ឋារចនាសម្ព័ន្ធ IoT និងប្រព័ន្ធ Cloud Computing ជាចាំបាច់។

  • Hardware (ផ្នែករឹង): ត្រូវការឧបករណ៍ចាប់សញ្ញា (Sensors), Actuators, និង IoT Gateways សម្រាប់ប្រមូលទិន្នន័យពីវាលស្រែ ឬកសិដ្ឋាន។
  • Connectivity (ការតភ្ជាប់): ការប្រើប្រាស់បច្ចេកវិទ្យា LPWAN (ដូចជា LoRaWAN) សម្រាប់ទីតាំងឆ្ងាយៗ និង Wi-Fi/Bluetooth សម្រាប់ការតភ្ជាប់ជិត។
  • Data Platform (ថ្នាលទិន្នន័យ): ចំណាយលើសេវាកម្ម Cloud ឬការដំឡើង Server ផ្ទាល់ខ្លួន (Local/Edge Computing) ដើម្បីរក្សាទុក និងវិភាគទិន្នន័យ។

៣. ការពិនិត្យសម្រាប់បរិបទកម្ពុជា/អាស៊ីអាគ្នេយ៍

ភាពលំអៀងនៃទិន្នន័យ (Data Bias)៖

ការសិក្សានេះវិភាគលើឯកសារស្រាវជ្រាវចំនួន ១៨ ដែលភាគច្រើនចេញពីប្រទេសអភិវឌ្ឍន៍ ដែលមានហេដ្ឋារចនាសម្ព័ន្ធបច្ចេកវិទ្យាខ្ពស់។ នេះអាចជាចំណុចខ្វះចន្លោះនៅពេលអនុវត្តនៅកម្ពុជា ដែលការតភ្ជាប់អ៊ីនធឺណិតនៅតំបន់ដាច់ស្រយាល និងចំណេះដឹងបច្ចេកវិទ្យារបស់កសិករនៅមានកម្រិត។

លទ្ធភាពនៃការអនុវត្ត (Applicability)៖

ស្ថាបត្យកម្មនេះមានសារៈសំខាន់ខ្លាំងសម្រាប់កម្ពុជា ក្នុងការរៀបចំប្រព័ន្ធកសិកម្មទំនើប (Smart Farming) ឱ្យមានស្តង់ដារ និងនិរន្តរភាព។

  • វិស័យដំណាំស្រូវ (ខេត្តបាត់ដំបង/ព្រៃវែង): អាចប្រើប្រាស់គំរូនេះដើម្បីរៀបចំប្រព័ន្ធគ្រប់គ្រងទឹក និងជី ដោយប្រើឧបករណ៍ចាប់សញ្ញា (Sensors) ដើម្បីកាត់បន្ថយថ្លៃដើម។
  • វិស័យដំណាំម្រេច និងឈើហូបផ្លែ (ខេត្តកំពត/កំពង់ចាម): ការប្រើប្រាស់ស្ថាបត្យកម្មនេះជួយក្នុងការបង្កើតប្រព័ន្ធស្រោចស្រពស្វ័យប្រវត្តិ (Smart Irrigation) ដែលផ្អែកលើទិន្នន័យជាក់ស្តែង។
  • បញ្ហាការតភ្ជាប់ (Internet Connectivity): ដោយសារស្ថាបត្យកម្មនេះណែនាំឱ្យមាន Local Gateway និង Edge Computing វាសាកសមនឹងតំបន់កសិកម្មនៅកម្ពុជាដែលសេវាអ៊ីនធឺណិតមិនទាន់គ្របដណ្តប់ពេញលេញ។

វាផ្តល់នូវក្របខ័ណ្ឌការងារច្បាស់លាស់សម្រាប់អ្នកអភិវឌ្ឍន៍បច្ចេកវិទ្យាកសិកម្ម (AgriTech) នៅកម្ពុជា ដើម្បីបង្កើតដំណោះស្រាយដែលអាចធ្វើការជាមួយគ្នាបាន (Interoperable) និងងាយស្រួលពង្រីក។

៤. ផែនការសកម្មភាពសម្រាប់និស្សិត (Actionable Roadmap)

ដើម្បីអនុវត្តតាមការសិក្សានេះ និស្សិតគួរអនុវត្តតាមជំហានខាងក្រោម៖

  1. ជំហានទី ១: ការយល់ដឹងពីស្រទាប់ទាំង ៤ នៃស្ថាបត្យកម្ម: និស្សិតត្រូវសិក្សាឱ្យច្បាស់ពីតួនាទីរបស់ស្រទាប់នីមួយៗ៖ Smart Product (ឧបករណ៍), Network (ការតភ្ជាប់), Data (ទិន្នន័យ), និង Smart Service (សេវាកម្ម)។
  2. ជំហានទី ២: ការអភិវឌ្ឍគំរូដើម (Prototyping): ចាប់ផ្តើមបង្កើតឧបករណ៍ Smart Product ដោយប្រើប្រាស់ Microcontrollers ដូចជា (ESP32) ឬ (Arduino) ភ្ជាប់ជាមួយសេនស័រសំណើមដី ឬសីតុណ្ហភាព។
  3. ជំហានទី ៣: ការរៀបចំបណ្តាញទំនាក់ទំនង: សាកល្បងប្រើប្រាស់បច្ចេកវិទ្យា (LoRaWAN) សម្រាប់ការបញ្ជូនទិន្នន័យចម្ងាយឆ្ងាយក្នុងកសិដ្ឋាន ដោយប្រើឧបករណ៍ដូចជា (Dragino Gateway)។
  4. ជំហានទី ៤: ការគ្រប់គ្រង និងវិភាគទិន្នន័យ: ដំឡើងថ្នាល IoT (IoT Platform) បែប Open-source ដូចជា (ThingsBoard) ដើម្បីប្រមូលផ្តុំទិន្នន័យ និងបង្កើត Dashboard សម្រាប់តាមដាន។
  5. ជំហានទី ៥: ការបង្កើតសេវាកម្មឆ្លាតវៃ (Smart Service): បង្កើតប្រព័ន្ធផ្តល់ដំណឹង (Notification System) ទៅកាន់កសិករតាមរយៈទូរស័ព្ទដៃ នៅពេលដែលទិន្នន័យបង្ហាញថាដំណាំត្រូវការទឹក ឬជី។

៥. វាក្យសព្ទបច្ចេកទេស (Technical Glossary)

ពាក្យបច្ចេកទេស (English) ការពន្យល់ជាខេមរភាសា (Khmer Explanation) និយមន័យសាមញ្ញ (Simple Definition)
Smart Farming ការប្រើប្រាស់បច្ចេកវិទ្យាព័ត៌មាន និងទំនាក់ទំនង (ICT) ទំនើបៗដូចជា IoT, Big Data, និង Cloud Computing ក្នុងវិស័យកសិកម្ម ដើម្បីបង្កើនផលិតភាព ប្រសិទ្ធភាព និងគុណភាព ដោយកាត់បន្ថយការប្រើប្រាស់កម្លាំងពលកម្មមនុស្ស។ ដូចជាការធ្វើឱ្យកសិដ្ឋានមាន «ខួរក្បាល» ដែលអាចប្រាប់កសិករថាដំណាំត្រូវការទឹក ឬជីនៅពេលណា ដោយមិនបាច់ដើរមើលផ្ទាល់។
Information System Architecture (ISA) ប្លង់មេ ឬគំរូដែលបង្ហាញពីរបៀបដែលផ្នែកផ្សេងៗនៃប្រព័ន្ធបច្ចេកវិទ្យា (ដូចជា ឧបករណ៍ចាប់សញ្ញា កុំព្យូទ័រ និងអ្នកប្រើប្រាស់) ធ្វើការតភ្ជាប់គ្នា និងផ្លាស់ប្តូរទិន្នន័យគ្នាទៅវិញទៅមក ដើម្បីសម្រេចគោលដៅរួមមួយ។ ដូចជាប្លង់សាងសង់ផ្ទះដែលកំណត់ថា បន្ទប់ទឹកនៅឯណា ខ្សែភ្លើងរត់យ៉ាងដូចម្តេច និងទ្វារនៅត្រង់ណា ដើម្បីឱ្យផ្ទះនោះអាចរស់នៅបានស្រួល។
Smart Connected Products ឧបករណ៍រូបវន្ត (ដូចជា ត្រាក់ទ័រ ឬម៉ាស៊ីនបូមទឹក) ដែលត្រូវបានបំពាក់ដោយសេនស័រ (Sensors) កម្មវិធី (Software) និងការតភ្ជាប់អ៊ីនធឺណិត ដែលអនុញ្ញាតឱ្យពួកវាអាចប្រមូលទិន្នន័យ និងបញ្ជាពីចម្ងាយបាន។ ដូចជាទូរស័ព្ទដៃធម្មតា (Phone) ក្លាយជាទូរស័ព្ទឆ្លាតវៃ (Smartphone) ដែលអាចធ្វើអ្វីៗបានច្រើនជាងការគ្រាន់តែហៅចេញ។
Low Power Wide Area Network (LPWAN) បច្ចេកវិទ្យាបណ្តាញឥតខ្សែដែលត្រូវបានរចនាឡើងដើម្បីបញ្ជូនទិន្នន័យក្នុងចម្ងាយឆ្ងាយ (រាប់គីឡូម៉ែត្រ) ប៉ុន្តែប្រើប្រាស់ថាមពលថ្មតិចតួចបំផុត ដែលស័ក្តិសមសម្រាប់ឧបករណ៍កសិកម្មនៅតាមវាលស្រែធំៗ។ ដូចជាការ «ខ្សឹប» ប្រាប់គ្នាក្នុងចម្ងាយឆ្ងាយ ដែលមិនធ្វើឱ្យអស់កម្លាំង (ឬអស់ថ្ម) លឿន ដូចការស្រែកខ្លាំងៗ (Wi-Fi/4G)។
Edge Computing ការដំណើរការ និងវិភាគទិន្នន័យនៅកៀកនឹងប្រភពទិន្នន័យ (ដូចជានៅលើឧបករណ៍ផ្ទាល់ ឬនៅកសិដ្ឋាន) ជាជាងការបញ្ជូនទិន្នន័យទាំងអស់ទៅកាន់អ៊ីនធឺណិត (Cloud) ដើម្បីកាត់បន្ថយភាពយឺតយ៉ាវ និងសន្សំសំចៃអ៊ីនធឺណិត។ ដូចជាការសម្រេចចិត្តបិទទឹកភ្លាមៗនៅពេលឃើញពាងពេញ ដោយមិនចាំបាច់ទូរស័ព្ទសួរម្ចាស់ផ្ទះសិន។
Smart Service សេវាកម្មដែលបង្កើតឡើងដោយផ្អែកលើទិន្នន័យពីឧបករណ៍ឆ្លាតវៃ ដើម្បីផ្តល់ជាដំណោះស្រាយ ឬការណែនាំដល់អ្នកប្រើប្រាស់ដោយស្វ័យប្រវត្តិ។ ដូចជាជំនួយការដែលផ្ញើសារប្រាប់អ្នកថា «សូមកុំអាលដាក់ជីនៅថ្ងៃស្អែក ព្រោះនឹងមានភ្លៀង» ដោយផ្អែកលើទិន្នន័យអាកាសធាតុ។
